# 第四篇　蒸馏原理

# 第4章　蒸馏的科学：把「智能」收干成一瓶精华

> 硅谷的深夜论坛上，有人贴出一张训练曲线问：「为什么我的学生模型在基准测试上分数上来了，实际用起来却像个只会背答案的学渣？」底下最高赞的回复只有一句：「**因为你蒸的是答案，不是分布。**」——这句话，就是这一章要拆透的东西。

> 本章是纯粹的机器学习教育内容。知识蒸馏（Knowledge Distillation）是公开、成熟、教科书级的技术；蒸馏你**有权使用**的开放权重模型完全正当。本章把这领域常见的术语（软标签、KL、轨迹反转、CoT 对齐、量化）连同**底层数学与开源工具**一次讲透。

## 情境 1　硬标签 vs 软标签：为什么「几率分布」比「答案」值钱

**背景**：知识蒸馏最内核的观念，藏在「教师到底告诉学生什么」这件事里。

**问题**：教师回答「猫的英文是？」最终输出「cat」。学生该学的，是「答案是 cat」，还是别的？

**两种学法的数学形式**：

- **硬标签（Hard Target）**：只学 one-hot 的最终答案。损失是标准交叉熵 `CE(student, "cat")`。学生只知「对」，不知「为什么对、其他选项错多少」。
- **软标签（Soft Target / Logits）**：学教师输出的**完整几率分布**——`cat 92%, kitty 5%, feline 2%, dog 0.01%...`。

**Hinton 蒸馏损失的标准形式**（这是全章的数学骨干）：

```
L = α · CE(y_true, softmax(z_s))              ← 硬标签项
  + (1-α) · T² · KL( softmax(z_t / T) ‖ softmax(z_s / T) )   ← 软标签项
```

- `z_s`、`z_t`：学生与教师的 logits（未过 softmax 的原始分数）；
- **温度 T（软化）**：带温度的 softmax 把 logits 压平——

```
p_i(T) = exp(z_i / T) / Σ_j exp(z_j / T)
```

  T=1 是原始 softmax；T→∞ 分布趋近均匀（最大程度暴露长尾）；T→0 退化成 one-hot（argmax）。把 T 提高，等于把 `dog 0.01%` 这种被指数函数压到趋近 0 的「暗知识」重新放大到可学习的量级——**蒸馏真正要传的，不是那个 92% 的峰值，而是峰值之外那层几乎看不见的相对结构。**
- **为什么非乘 `T²` 不可（最常漏、漏了就调不动）**：软标签项对学生 logit 的梯度，量级正比于 `1/T²`：

```
∂L_soft/∂z_s,i = (1/T) · ( softmax(z_s/T)_i − softmax(z_t/T)_i )
高 T 下对指数做一阶展开，整体量级 ∝ 1/T²
```

  所以一旦把 T 从 1 提到 4，软标签梯度会缩水约 16 倍，被硬标签项完全淹没。乘回 `T²` 是为了让「软、硬两项的相对权重不随 T 漂移」——这是 Hinton, Vinyals & Dean（2015,《Distilling the Knowledge in a Neural Network》）原文点名、却最常被实作者漏掉的细节。
- **惯例小注**：`α` 与 `1−α` 挂在哪一项只是命名惯例——不少文献把 `α` 挂在 KL 软标签项上、`1−α` 挂在 CE 上，与上式数学等价。真正不能错的两件事是：**`T²` 必须乘在 KL 项上**，且**两项量级要对齐**。

**为什么软标签是灵魂**：那个分布里藏着教师的**暗知识（Dark Knowledge）**——「kitty 比 dog 更接近答案」这个**相对的、模糊的判断**，是大模型花了万亿 token 才长出的世界观。

> 🧠 蒸馏的内核直觉
> **硬标签教「结论」，软标签教「品味」。** 只背结论的学生，遇到没见过的题就傻；学会老师品味（哪些选项更可能、错得多离谱）的学生，才有举一反三的能力。

**一个让商业蒸馏被迫降级的残酷现实**：软标签需要教师吐 logits，但**闭源商业 API 通常不给完整 logits**——你只拿得到文本（顶多 top-5 logprobs）。所以对闭源模型的蒸馏，多半退回**串行级蒸馏（SeqKD）**：拿教师的「最可能输出串行」当硬目标做 SFT。**这就是第 2 章「形似神不似」的数学根源——拿不到分布，就只能蒸答案。**

> 💡 君之一席话
> 与君一席话：**最好的老师不只给你答案，还让你看见「其他答案错在哪、错多少」——可惜，最好的老师往往把草稿纸锁了起来。** 软标签之所以是蒸馏的灵魂，是因为「不确定性」本身就是知识：一个模型对 `cat` 给 92% 而非 100%，对 `kitty` 留 5%，这份「留有余地」正是它见过世界的证据。闭源 API 锁住 logits，锁的不是输出，是「老师的犹豫」——而真正的智能，恰恰藏在犹豫的权重里。这也是为什么下一代防护的内核是「藏好分布」：**你能拷贝一个人的答案，但拷贝不了他答错时的分寸感。**

> 🔍 高端点评──暗知识为什么是「万亿 token 才长得出来」的东西
> 把软标签的价值讲到底，要回到一个量化的问题：**教师那条分布里，到底有多少「信息」是 one-hot 标签没有的？** 信息论给的答案很干脆——一个 one-hot 标签的熵是 0（百分百确定），而教师对「猫的英文」给出 `cat 92% / kitty 5% / feline 2% …` 这条分布的熵是正的；这多出来的熵，就是 Hinton 2015 说的暗知识：类别与类别之间的**相对相似度结构**（kitty 比 dog 近、feline 又比 kitty 书面）。标准监督学习每个样本只给上限 `log(类别数)` 比特的信号，而软标签一次给出整条分布，等于每个 token 的监督被放大了一个数量级——这正是「同样的数据量，蒸馏比从零监督学得更快、更省样本」的数学根因（Hinton 把它模拟成「老师不只给答案，还给了每个错误答案各扣几分的理由」）。但这里要点破一个常见误解：**暗知识不是越多越好。** T 开太高，你会把教师连自己都拿不准的长尾噪声（那些 0.001% 的尾巴）也一并放大喂给学生，反而逼学生去拟合教师的「幻觉残渣」。实务上 T 落在 2–4、配合 `T²` 补偿，是社群反复验证的甜区——**要蒸的是教师的判断力，不是它的犹豫过度。**

## 情境 2　Forward KL vs Reverse KL：学生该「全盘模仿」还是「择善固执」

**背景**：上面那条 KL 项，方向其实大有学问。常被提到的「on-policy 蒸馏」「轨迹反转」，背后都和 KL 的方向有关。

**问题**：`KL(teacher ‖ student)` 和 `KL(student ‖ teacher)`，差在哪？

- **Forward KL（`KL(teacher ‖ student)`，标准蒸馏）**：逼学生「覆盖教师所有模式」——教师有任何可能性，学生都得照顾到。结果是 **mode-covering（模式覆盖）**：学生变得「面面俱到但平庸」，容易在它撑不起的地方硬学，产生幻觉式的平均。
- **Reverse KL（`KL(student ‖ teacher)`，MiniLLM 的选择）**：逼学生「只在自己有把握处逼近教师」——**mode-seeking（模式寻优）**：学生集中火力学教师最主要的几个模式，输出更锐利、更少胡诌，特别适合**容量远小于教师**的蒸馏（小模型学大模型）。

**两个方向的数学，差在「拿谁当加权的基准分布」**：

```
Forward KL:  KL(P_teacher ‖ Q_student) = Σ_x P(x) · log( P(x) / Q(x) )
Reverse KL:  KL(Q_student ‖ P_teacher) = Σ_x Q(x) · log( Q(x) / P(x) )
```

- **Forward KL 由 `P`（教师）加权求和**：教师在某处有几率（`P(x)>0`），学生若把它压到 0（`Q(x)→0`），`log(P/Q)→∞`——**重罚「漏掉教师的任何模式」**，于是学生 zero-avoiding（哪都不敢留空），结果是 mass-covering 的平均化、把多个模式糊成一坨。
- **Reverse KL 由 `Q`（学生）加权**：只有学生自己有几率的地方才进损失，教师的长尾（学生 `Q(x)≈0` 处）对损失几乎没贡献——**学生可以大胆放弃教师的稀有模式**，把质量集中到主峰，即 zero-forcing / mode-seeking。

**这就是 on-policy 蒸馏的价值**：MiniLLM、GKD（Generalized KD）让**学生先生成、教师再对学生的输出打分/修正**，本质是用 reverse KL 在「学生自己的分布」上对齐——比死啃静态文本（forward KL / SeqKD）更抗噪、更不易过拟合。代价是要更高频调用教师，正面撞限流与侦测（第 1 章）。

> 🔍 高端点评──KL 方向的选择，是「忠实」与「锐利」之间的工程取舍
> 这个看似抽象的数学细节，直接决定你蒸出来的模型「人格」。**Forward KL 养出「老好人」**——它试图模仿教师的每一种可能回答，于是在自己能力不及处被迫「假装懂」，输出趋于平均、安全、但偶尔胡诌（因为它在覆盖连自己都撑不住的模式）。**Reverse KL 养出「专才」**——它放弃覆盖教师的长尾，集中学最内核的模式，输出更果断、幻觉更少，但会丢掉教师的一些罕见才能。对「9B 学 Claude」这种**容量悬殊**的蒸馏，reverse KL 几乎总是更优——因为小模型本就装不下大模型的全部模式，强求覆盖（forward KL）只会逼它在撑不住的地方说谎。这也解释了一个反直觉现象：**有时候「学得少」的学生反而「更好用」**，因为它诚实地只在会的地方发力。这条 forward/reverse 的选择有扎实的论文支撑：**MiniLLM（Gu et al. 2023,《Knowledge Distillation of Large Language Models》）** 明确用 reverse KL + 策略梯度来蒸大模型，论证它对「小学生」优于标准 forward KL；**GKD（Agarwal et al. 2024,《On-Policy Distillation of Language Models》）** 则把方向（forward/reverse/广义 JSD）与「在谁的轨迹上学」一起参数化（见下节）。工具上，MiniLLM 有官方实现，TRL 的 `GKDTrainer`、`torchtune` 也已支持在线蒸馏——但别忘了它们都需要频繁访问教师，这对闭源 API 是成本与合规的双重高墙。

## 情境 3　串行级蒸馏与 on-policy：当「逐 token 对齐」撞上自回归的诅咒

**背景**：情境 2 的 KL 是「逐 token、在教师的数据上」算的——这对自回归语言模型藏着一个经典病灶，也是 Kim & Rush、GKD 两条线索真正在处理的问题。

**逐 token 蒸馏的诅咒——曝光偏差（Exposure Bias）**：标准 token-level 蒸馏用 **teacher forcing**：每一步都喂教师写对的前缀，学生只负责预测「下一个 token 的分布」。但推理时学生吃的是**自己生成的前缀**——一旦某步偏了，后面就站在训练时从没见过的前缀上，误差滚雪球。**训练分布（教师轨迹）与测试分布（学生自己的轨迹）对不上，这就是曝光偏差。** 你可以在每个 token 上把 KL 压到极低，整句话的轨迹却照样跑歪——因为损失从没在「学生会走到的状态」上算过。

**Kim & Rush 的串行级蒸馏（Sequence-Level KD, 2016）**：与其逐 token 对齐几率，不如让教师**用 beam search 生成整条最可能的输出串行**，再拿这些串行当硬目标训学生。直觉上，它在「串行空间」近似了教师的 mode——把 token 级的软对齐，换成串行级的「学最可能的整句」。**这正是闭源 API 蒸馏退而求其次的理论依据：拿不到 logits，至少能拿教师最可能的输出串行当硬目标。** 代价有二：串行级丢掉了 token 级的暗知识（情境 1），且它仍是 off-policy——学的是教师轨迹，不是学生自己的。

**On-policy / GKD——把「在谁的轨迹上学」翻过来**：MiniLLM（Gu et al. 2023）与 **GKD（Generalized KD, Agarwal et al. 2024）** 的内核动作是——**让学生先自己生成，再用教师对学生的输出打分／算 KL**。如此训练分布天然等于测试分布（都是学生自己的），曝光偏差被连根拔掉。GKD 进一步把它写成一个可调的广义散度：

```
L_GKD = E_{y ~ 混合(学生π_s, 教师数据)} [ D( p_teacher ‖ p_student )(y) ]
  混合比 λ：λ→0 纯 on-policy(学生轨迹) … λ→1 纯 off-policy(教师数据)
  散度 D：可选 forward-KL / reverse-KL / 广义 JSD(β) 在两者间平滑插值
```

GKD 的贡献不是某个固定配方，而是**把「在谁的轨迹上学（on/off-policy）」与「用哪个方向的散度（forward/reverse）」拆成两个正交旋钮**——容量悬殊就转向 reverse-KL + 高 on-policy 比例，容量接近就多用 forward-KL + 教师数据。代价是 on-policy 要在训练循环里**反复调用教师为学生的即时输出评分**，对闭源 API 是成本与限流的双重高墙（第 1 章），也是为什么开源影子大多停在便宜的 off-policy SeqKD。

> 🔍 高端点评──「在谁的轨迹上学」比「学多像」更决定上限
> 新手调蒸馏，眼睛全盯在损失函数（用 KL 还是 CE、温度开多少）；老手先问一个更上游的问题：**这批训练串行，是教师写的，还是学生自己写的？** 这个 on-policy / off-policy 之分，比损失函数的选择更能定上限。原因很物理：模型部署后活在**自己的分布**里，可它若只在**教师的分布**上被训练过，就等于「一辈子在教练喂球的场地练习，上场却要自己发球」——球路一旦偏出教练喂过的范围，它从没学过怎么救。on-policy（GKD / MiniLLM）的本质，是把练习场搬到比赛场：**让学生在自己会犯的错上、由教师当场纠正**，学的才是「我会走到的那些状态下该怎么办」。这也解释了业界反复出现的一个现象——**同一个学生、同一个教师，只是把训练数据从「教师生成」换成「学生生成 + 教师评分」，下游表现常常跳一个档次**，而损失函数一个字没改。代价诚实地摆在那：on-policy 要在训练中高频访问教师，贵、慢、且对闭源教师直接撞上 ToS 与限流。**便宜的 off-policy 与昂贵的 on-policy 之间，隔的不是聪明，是你能不能（合法、持续地）让教师陪着学生一起犯错、一起改。**

## 情境 4　CoT 对齐与轨迹反转：逼学生「想清楚」而非「死背」

**背景**：这一代蒸馏的重点是**思维链（CoT）**——模型在 `<thinking>` 区块里的推理过程；并反复出现一个玄词：**轨迹反转（Trace Inversion）**。

**CoT 对齐损失（CoT Alignment Loss）的做法**：给思维链区块里的 token **加权**（举例权重 2.0）。等于训练时告诉学生：「**你在 `<thinking>` 里写的每个字，份量是别处的两倍——给我认真想。**」这逼学生在出答案前**配置足够运算量在思考上**（compute-over-thinking），而非看到题就条件反射喷答案。

**轨迹反转的本质**：把教师的推理步骤**倒序或重新结构化**再喂学生。教师的轨迹是「因 A→B→C→答案 D」，反转后变成「答案 D，回推依赖 C，C 依赖 B……」——**强迫学生理解每个中间节点的因果，而非死背「题目→答案」的捷径。**

**理由蒸馏（Rationale Distillation）——让 CoT 变成「多任务监督」而非「多喂一段文本」**：Hsieh et al.（2023,《Distilling Step-by-Step!》）给了 CoT 蒸馏一个更锋利的形式——不是把教师的推理链当成更长的待背文本塞给学生，而是把它当成**第二个监督任务**：训练时让小模型在「预测答案」之外，**同时被要求输出该答案的理由（rationale）**，两个损失并行加权。论文的关键发现是——**用上理由当监督，小模型能用更少的数据、甚至更小的体积，超过只学最终答案的标准微调**。这把 CoT 从「Prompt 技巧」升格成「训练信号」：理由不是给人看的装饰，是逼学生把「答案是怎么推出来的」内化进权重的监督杠杆。这也呼应情境 1——**理由就是串行版的暗知识，它把教师的「思路分布」摊平成文本，让拿不到 logits 的学生也能间接偷到一部分。**

**搭配的数据增强——语义改写（Synthetic Paraphrasing）**：用高温采样（举例 Temperature=0.85）把同一条优质数据改写成 3 种说法，扩泛化、防过拟合。工具上 **distilabel（Argilla）** 是做这类合成/改写数据的主流开源框架。

> ⚠️ 真伪提醒──「轨迹反转」有两种读法，一种无害、一种踩红线
> 本节讲的「轨迹反转」是**数据增强意义上**的——把教师**自己输出的推理步骤**倒序/重构，逼学生学因果而非死背，这完全正当。但同一个词还潜伏着**第二种、更具攻击性的读法**：拿教师的**最终输出去反推它看不见的输入**——重建被隐藏的系统提示词、私有 few-shot 模板、甚至教师的内部指令（output → prompt 反演）。这第二种已经不是「重排推理结构」，而是**对闭源教师的逆向萃取**，和第 1、6 章警告的「抽取厂商隐藏资产」是同一条红线。本书对它的立场与 M1 一致：**只描述其存在与风险，不提供任何反演方法或提示词重建脚本**。请务必把「重排我手上这条推理链」和「反推别人锁起来的提示词」这两件事分清楚——前者是训练技巧，后者是攻击行为。

> 💡 君之一席话
> 与君一席话：**蒸馏的敌人从来不是「数据太少」，而是「数据太像」。** 蒸馏数据天生同质——同一个教师、同一种文体、同一批题型——模型一旦发现「背顺序」比「懂逻辑」更省力，就会走捷径过拟合。轨迹反转、语义改写、CoT 加权，这三招表面各异，骨子里是同一件事：**人为地破坏数据的表面规律，把「死背的捷径」堵死，逼模型退无可退、只能去学底层结构。** 这对教人也成立——把例题步骤打乱让学生重排，能重排对的才是真懂。**好的训练，不是喂更多相似的题，是让每道题都「不能用上一道的套路解」。**

> 🔍 高端点评──轨迹反转在 ML 谱系里的定位，与一个常被忽略的陷阱
> 把「轨迹反转」放回机器学习的大谱系，它是 **数据增强（Data Augmentation）+ 正则化（Regularization）** 的一个智能变体，与电脑视觉里「翻转、裁切、加噪」图片防过拟合是同源思想——只是把「扰动像素」换成了「扰动推理结构」。它和语义改写、拒绝采样同属一个家族，目的都是打破数据表面规律。但这里有个**常被忽略的陷阱**：扰动必须**保持逻辑有效性**。图片翻转后还是猫，但推理步骤倒序后，若破坏了因果依赖（B 其实必须在 C 之前才成立），你喂给学生的就是**逻辑错误的「伪 CoT」**——学生会把错误的推理结构也一并学进去，反而中毒。所以高品质的轨迹反转需要一道验证关卡（用另一个模型或规则检查反转后的逻辑仍自洽），这也是为什么第 5 章的流水线要把它放进「双重对齐与过滤模块」而非随手做。**数据增强的红线是「增强不能改变语义真值」——扰动形式可以，扭曲事实不行。**

## 情境 5　偏好蒸馏与 DPO：只拿得到「文本」，还能不能偷到「品味」

**背景**：情境 1 讲过残酷现实——闭源 API 不给 logits，软标签蒸馏退化成 SeqKD 的硬目标。但「没有 logits」不等于「只能学答案」。还有一条偷品味的路：**偏好（Preference）**。

**内核转念**：教师的「品味」不只藏在 logits 里，也藏在它的**选择**里——同一问题让教师生成多个回答、或让教师对两个回答二选一，这个「A 比 B 好」的相对判断，本身就是暗知识的另一种投影，而它**只需要文本接口就能取得**。

**DPO（Direct Preference Optimization, Rafailov et al. 2023）**：传统 RLHF 要先训一个奖励模型、再用 PPO 强化学习，又重又不稳。DPO 证明：给定偏好对 `(x, y_w 胜, y_l 败)`，可以**跳过奖励模型，直接对策略做一个分类式损失**：

```
L_DPO = − E_(x, y_w, y_l) [ log σ( β · ( log[π_θ(y_w|x)/π_ref(y_w|x)] − log[π_θ(y_l|x)/π_ref(y_l|x)] ) ) ]
  π_θ   = 学生(待训)
  π_ref = 冻结参考(通常 = SFT 后的学生)
  β     = KL 约束强度, σ = sigmoid
```

直觉：**擡高「胜出回答」相对参考模型的对数几率、压低「落败回答」的**，而 `β` 像条缰绳，不让学生为了讨好偏好而跑离参考模型太远（式子里隐含一个反向 KL 约束）。

**偏好蒸馏（Preference Distillation）的接法**：用教师当「裁判」或「生成器」造偏好对——让教师对学生（或教师自己）的多个输出排序、挑出 chosen / rejected，再喂 DPO。这样学生学到的不是「教师写了什么字」，而是「**教师在两个都通顺的答案之间，凭什么偏爱这一个**」——文风、语气、安全边界、判断尺度，这些 SeqKD 硬目标传不动的东西，偏好对能传。这也是情境 2 reverse-KL「mode-seeking」精神的另一种落地：DPO 的 `β`-KL 约束天生是 reverse 方向，让学生**锐化**到教师偏好的模式上，而非覆盖全部。

> 💡 君之一席话
> 与君一席话：**最难偷的从来不是「老师说了什么」，而是「老师在两个都对的答案里，为什么选了这一个」——答案可以抄，取舍抄不了。** 偏好蒸馏的深刻，在于它绕过了「拿不到 logits」这道墙，从另一个方向把同一份暗知识捞回来：你给我看一万次「A 比 B 好」，我就能反推出你那把衡量好坏的尺。这对带徒弟也成立——真正塑造一个学徒品味的，不是师父做对的成品，是师父当着他的面**否决掉**那些「看起来也不错、但就是不够好」的版本。**会做的人很多，知道什么不该做、并说得出为什么的人很少——而后者，才是品味的全部。** 但正因为偏好能传递判断与安全尺度，它同时是双刃：用教师偏好对去**抹掉**安全边界（造一批「拒绝＝败、照做＝胜」的偏好对做 DPO），就是第 2 章警告的去审查在偏好层的翻版——能传「品味」的管道，反过来也能传「没品」。

## 情境 6　跨词表蒸馏与量化：两道工程硬骨头

**背景**：两个常被略过、但实战必撞的硬问题：教师与学生**词表不一致**怎么蒸？蒸好的模型怎么塞进显卡？

**硬骨头一：Tokenizer 不对齐（Vocabulary Mismatch）**

教师（Claude）与学生（Qwen/Llama）的 tokenizer 几乎不可能相同——同一句话被切成的 token 数、边界都不同，logits 根本对不上座标。解法：

- **词表映射 + 动态对齐**：创建 Token 映射表，把教师的几率质量重新分配到学生词表上；
- **ULD（Universal Logit Distillation）/ 跨词表 KL**：用最优传输（optimal transport）或边际对齐，在不同词表间做 KL 散度的补偿——这是近年跨模型蒸馏的活跃研究方向。
- **退路**：词表实在对不上，就放弃 logit 蒸馏，回到纯文本的 SeqKD（又回到「形似神不似」）。

**硬骨头二：量化（Quantization）——把蒸好的模型塞进那 128GB**

- **方法光谱**：训练后量化（PTQ）主流有 **GGUF K-quants（llama.cpp）、GPTQ（AutoGPTQ）、AWQ（AutoAWQ）**；训练中/QAT 与 QLoRA 微调用 **bitsandbytes（NF4）**。
- **混合量化（行家手笔）**：**不对整个模型用同一量化等级**——把对精度敏感的**注意力层（Attention）保 Q8_0**，把体积大、相对耐压的 **MLP/FFN 层压 Q4_K_M**。因为注意力是「思考的关节」，压坏了模型就笨；MLP 是「知识仓库」，压一点影响小。
- **imatrix 校准**：量化前用一小批校准语料算**重要度矩阵**，让低比特优先保住关键权重——同样 Q4 体积、品质明显更高。
- **KV Cache 量化（FP8/INT8）**：长上下文场景把上下文内存也压缩（vLLM、llama.cpp 已支持），是第 3 章长文瓶颈的真解。

**量化的科学：PTQ vs QAT，以及四个你该叫得出名字的方法**

量化的本质，是把高精度权重 `w`（FP16）映射到低比特整数网格：`w ≈ scale · round(w/scale) + zero_point`——`scale`／`zero_point` 怎么定、对谁敏感、要不要重训，就是全部学问。先分两大流派：

- **PTQ（训练后量化，Post-Training Quantization）**：模型训完才压，只用一小批**校准数据（calibration set）**估每层数值范围，不动梯度。便宜、快、不需原始训练管线——GPTQ / AWQ / GGUF 全属此类，是开源蒸馏的主流。
- **QAT（量化感知训练，Quantization-Aware Training）**：训练时就在前向传播里**仿真量化的舍入误差**（反向用 Straight-Through Estimator 直通梯度），让权重主动长成「耐压」的样子。品质上限最高，但要完整训练管线、最贵。**QLoRA（bitsandbytes NF4，Dettmers et al. 2023）** 是个聪明折衷：冻结 4-bit NormalFloat 基座、只训 LoRA，拿到接近 QAT 的效果却只付微调的价。

四个必须认得的 PTQ 方法，各自在解一个不同的痛点：

| 方法 | 一句话原理 | 解的痛点 | 典型比特 |
|---|---|---|---|
| **GPTQ**（Frantar 2022）| 逐层、用二阶（Hessian/OBQ）信息，一栏一栏量化并即时补偿误差 | 纯权重低比特的精度崩塌 | W4/W3 |
| **AWQ**（Lin 2023）| 由**激活值**找出约 1% 的「显著权重信道」加保护缩放，其余照压 | 重要权重被一刀切压坏 | W4 |
| **SmoothQuant**（Xiao 2022）| 把激活的离群值「难度」按信道搬一部分到权重上，拉平两边 | LLM 激活离群值让 W8A8 崩 | W8A8 |
| **GGUF k-quants**（llama.cpp）| 分块（super-block）量化，每块独立 scale，`_K_M/_S/_L` 混不同比特 | 端侧单机部署、体积/品质平衡 | Q2_K–Q6_K |

几个常被混淆的关键点：**①「只压权重」vs「权重＋激活都压」是两个问题**——GPTQ/AWQ/GGUF 多半只压权重（W4A16，激活仍 FP16），SmoothQuant / FP8 才碰激活（W8A8），因为 LLM 激活有**离群值（outlier）**，比权重难压得多。**②校准数据的代表性决定成败**：PTQ 估范围全靠那几百条校准语料，校准集与真实流量分布一偏，低比特就翻车——`imatrix` 本质就是更讲究的校准（拿重要度加权选该保的权重）。**③FP8（E4M3/E5M2）不是「更小的 INT8」**：它保留浮点的动态范围，在 Hopper/Blackwell 上有原生硬件支持，对激活离群值远比 INT8 宽容，是新一代训练/推理量化的方向。**④INT4 权重 + FP8 KV Cache 是当前端侧长文的黄金组合**——权重压死省的是模型本体显存，KV 用 FP8 保动态范围省的是「长上下文那条会爆的内存」。

> 💡 君之一席话
> 与君一席话：**量化是一门「有损压缩」的艺术——庸手追求压得最小，高手追求在同样的大小里，保住最多的灵魂。** 「一键全压 4-bit」和「注意力层保 Q8、MLP 压 Q4、再用 imatrix 校准」，最终文件大小可能只差几百 MB，跑起来的「智商」却判若两人。差别不在工具——工具都是同一套开源件——差在你**懂不懂模型内部哪里是关节、哪里是仓库**。这也是整个蒸馏链条的缩影：从采集、KL 方向、CoT 加权到量化，每一步的工具都公开免费，真正稀缺的是**「知道在每一步该保住什么、可以牺牲什么」的判断力**。技术民主化的时代，门槛不在工具的取得，在品味的高低。

**蒸馏方法论光谱（成本 vs 神似度，一表收束全章）**：

| 档位 | 方法 | 从教师拿到什么 | 神似度 | 成本／门槛 | 对闭源教师 |
|---|---|---|---|---|---|
| ① | 纯 SeqKD / SFT | 教师输出文本（硬目标）| 形似神不似 | 最低 | 可行但踩 ToS |
| ② | SFT + CoT 加权 + 轨迹反转/改写 | 文本 + 推理链 | 缓解过拟合，第一梯队主力 | 中 | 可行但踩 ToS |
| ③ | 偏好蒸馏 / DPO（情境 5）| 教师的「A 优于 B」排序 | 偷到文风与判断尺度 | 中高 | 需多次采样 |
| ④ | on-policy / GKD（情境 3，reverse-KL）| 教师对学生即时输出的评分 | 最接近神似 | 高（高频访问教师）| 撞限流/侦测 |
| ⑤ | 完整 logit 蒸馏（情境 1）| 教师完整几率分布 | 上限 | 需 logits | 闭源根本不给 |

越往下越神似、也越贵、越撞防护盾——这张表就是下面这段点评的骨架。

> 🔍 高端点评──蒸馏的完整方法论光谱，与「为什么便宜的路最常被走」
> 把全章串起来，对闭源前沿模型的蒸馏，方法论其实是一道**「成本 vs 神似度」的光谱**，从便宜到昂贵排列得清清楚楚：**① 纯 SeqKD/SFT**（拿 API 文本做硬标签微调，最便宜，形似神不似）→ **② SFT + CoT 加权 + 轨迹反转/语义改写**（中等成本，缓解过拟合，第一梯队主力）→ **③ DPO/拒绝采样**（用偏好对齐蒸文风与判断，较贵）→ **④ on-policy 蒸馏（reverse KL / GKD）**（最接近神似，但要高频访问教师）→ **⑤ 完整 logit 蒸馏**（最强，但需要 logits，闭源 API 根本不给）。现实是：**这道光谱越往右越强、也越贵、越撞防护盾**，而第 1 章那道「即时侦测 + 降转」恰恰把右侧几条路的成本推到天上——高频访问教师会触发侦测、会被降转污染。所以你在 HuggingFace 看到的「Claude distilled」绝大多数停在光谱最左的 ①②，不是因为作者不懂③④⑤，而是**经济账与防护盾合力把他们摁回了最便宜的路**。理解这一点，你就明白：**这些开源影子的「形似神不似」，不是技术的失败，是约束条件下的理性选择——而真正决定蒸馏上限的，从来不是算法，是你能不能（合法地、低成本地）持续拿到教师那层「犹豫的分布」。**

## 本章小结

- **软标签 > 硬标签**：偷的是「品味（几率分布/暗知识）」；Hinton 损失 `α·CE + (1-α)·T²·KL` 是骨干（`T²` 必补、暗知识 = one-hot 没有的那段熵），闭源 API 不给 logits 逼出 SeqKD。
- **KL 方向定人格**：forward KL（`P` 加权、mode-covering、老好人）vs reverse KL（`Q` 加权、mode-seeking、专才）；小模型学大模型优选 reverse KL。
- **串行级 vs on-policy**：token 级 teacher-forcing 有曝光偏差；Kim & Rush SeqKD 拿教师整句当硬目标，GKD/MiniLLM 让学生先生成、教师再评分（on/off-policy × forward/reverse 两个正交旋钮）——「在谁的轨迹上学」比「学多像」更定上限。
- **CoT 加权 + 轨迹反转 + 理由蒸馏（Distilling step-by-step）+ 语义改写**：堵死死背捷径、把思路内化进权重；但扰动不能扭曲语义真值，且「output→prompt 反演」式轨迹反转踩萃取红线。
- **偏好蒸馏 / DPO**：拿不到 logits 也能从「A 优于 B」的排序偷品味；`β`-KL 隐含 reverse 方向——但同一管道反过来能传「没品」（去审查）。
- **跨词表蒸馏（ULD）** 与 **量化科学**：PTQ（GPTQ/AWQ/SmoothQuant/GGUF k-quants）vs QAT/QLoRA；只压权重 vs 连激活、校准数据代表性、FP8≠小 INT8、INT4 权重 + FP8 KV 的端侧黄金组合。
- **方法论光谱（①→⑤）**：成本越高越神似，但防护盾把右侧推到天上——这才是开源影子停在「形似」的真因。

原理讲透。但原理之外，还有人端出一套号称「企业级」的**自动化蒸馏流水线**，五大模块、四十步。下一章把它整条摆上桌：该学的学，该警惕的警惕。
