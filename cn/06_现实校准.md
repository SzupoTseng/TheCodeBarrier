# 第六篇　现实校准

# 第6章　那条线有多红：法律、伦理与一笔算到最后的帐

> 硅谷，律师事务所的玻璃会议室。一份「用蒸馏模型加速产品」的提案被推回到桌子中央，法务只问了一句：「你们训练数据里，有没有一条来自别人家的付费 API？」会议室安静了三秒——那三秒的沉默，比这一整章任何一句话都值钱。

> 这是全书最重要的一章。前五章拆「能做什么」，这一章谈「该不该做」与「划不划算」。它不是道德说教，而是一个工程师在按下 Enter 之前，必须替自己和公司算清的那笔帐。

## 情境 1　法律与条款：四条你必须先知道的事实

**背景**：把坊间流传的那些零散警告——「违反 ToS」「推动定为刑事犯罪」「指控阿里巴巴」——收拢成能用的判断依据。

**四条事实（基于公开数据与常识，非法律意见）**：

1. **ToS 几乎必然禁止蒸馏**。主流商业 LLM 厂商的条款，普遍明文禁止「用我家模型的输出去训练竞品模型」。大规模采集 API 响应拿去训练，**违约近乎确定**，后果从封号、追偿到诉讼。
2. **「绕过防护」可能让事情从「违约」升级为「违法」**。单纯违反 ToS 通常是民事问题；但若你**主动绕过技术防护措施**（第 5 章 M1 那套代理轮换、对抗提示词、降级侦测），在某些法域可能触及「规避技术保护措施」「未授权访问电脑系统」这类**刑事或准刑事**条款。坊间流传的「推动将大规模蒸馏攻击定为刑事犯罪」，正是这个方向。
3. **数据集本身带「法律放射性」**。下载一份来路不明的 `claude-traces`，即使不是你采集的，**使用它训练商业模型，仍可能让你成为侵权链条的一环**（第 2 章）。
4. **企业场景风险被放大 N 倍**。个人玩票被抓可能就封号；公司被认定靠蒸馏竞品创建商业模型，面对的是**商业诉讼、赔偿、品牌信任崩塌**——量级完全不同。

> ⚠️ 免责
> 以上不是法律意见。真要在商业场景碰这条线，**请咨询专业律师**。本书立场很简单：**对闭源商业模型的未授权蒸馏，技术可行，法律与伦理不可取。**

> 🔍 高端点评──为什么「绕过防护」是整件事里法律性质最严重的一步
> 第 2 条值得单独深掘，因为它是多数工程师最容易低估的法律断层。**违反 ToS 与绕过技术防护，是两种完全不同量级的事。** 前者本质是「违约」——你和厂商有合约，你违反了使用条款，这通常落在民事范畴（封号、求偿）。但当你部署代理 IP 池去规避限流、用对抗提示词去骗过防护分类器、用降级侦测去对抗厂商的污染机制时，你做的已经不是「违反一份合约」，而是「**主动破解一个为阻止你而设的技术控制系统**」——这在许多法域（如美国 CFAA「未授权访问」、DMCA「规避技术保护措施」的相关讨论，及各国类似立法）会把行为的法律性质从民事推向刑事。**这就是 M1 为什么是整条流水线里最危险的一块**：它不只让你「违约」，它让你的每一次采集都成为「规避控制」的证据。一个关键的判断启示：**当你发现自己需要「骗过」或「绕过」对方专门设来阻止你的机制时，你大概率已经从「灰色地带」迈进了「黑色地带」——技术上的「能绕过」，恰恰是法律上「故意性（intent）」最有力的证明。** 防护盾的存在本身，就是一块写着「禁止进入」的牌子；绕过它，你就再也无法主张「我不知道这不被允许」。

### 条款的结构：三大厂商「禁止什么」其实高度一致

别把「ToS 禁止蒸馏」当成一句模糊的口号——它在每一家主流厂商的合约里，都对应到一条**结构几乎相同的「禁止以输出训练竞品」条款**。把三家摆在一起看，你会发现这不是某一家的脾气，而是整个产业的共识防线：

| 厂商（条款所在）| 普遍包含的限制类型 | 对蒸馏者的意义 |
|---|---|---|
| OpenAI（Usage Policies / Business Terms）| 不得用「输出」开发与其竞争的模型 | 拿输出训练学生 = 直接命中 |
| Anthropic（Usage Policy / Commercial Terms）| 不得用服务创建竞品，**含训练竞争性 AI 模型** | 蒸馏 Claude 系列 = 直接命中 |
| Google（Gemini API 附加条款 / 禁用政策）| 限制以服务输出开发竞争性 ML 模型 | 蒸馏 Gemini = 直接命中 |

> ⚠️ 真伪提醒
> 上表是**结构性归纳，不是逐字引用**，且各家条款**版本更迭频繁**（近年三家都调整过相关措辞，部分产品线放宽、部分收紧）。真要判断，**以你签约当下、该产品线的最新条款原文为准**，并请法务确认。本书要你记住的不是某一句话，而是——**那条防线一直在那里，三家都在。**

**一个多数工程师没想清楚的法律断层：这主要是「合约问题」，不是「著作权问题」。** 美国著作权局已多次表明：**缺乏人类创作的纯 AI 生成内容，不具可注册的著作权**——也就是说，教师模型吐出的原始 token，本身**多半不受著作权保护**。这很容易让人误判成「那拿来用就没事」，真相却恰恰相反：**正因为著作权这道墙可能不存在，厂商的保护主要创建在你按下「同意」时生效的那纸合约（ToS）之上**，再叠加可能的**营业秘密**与前述**未授权访问／规避防护**等电脑犯罪法理。换句话说，蒸馏闭源模型的法律风险，**不取决于输出有没有著作权，而取决于你有没有违约、有没有绕过控制**——这也是为什么「我只是用了它公开吐出来的文本」这种辩解，在合约与 CFAA 一类法理面前几乎没有意义。

> ⚠️ 真伪提醒
> 「AI 输出能否受著作权保护」「拿模型输出去训练另一个模型是否构成侵权」目前**全球都没有定论**——各国立场分歧、诉讼正在进行，而且「训练数据端的争议」与「模型输出端的争议」是**两个不同问题**，别混为一谈。本段陈述的是**目前的开放问题与主流方向**，不是已定案的法律。

### 两道正在升起的新墙：监管与出口管制

红线不只来自厂商合约，还来自国家。两个方向值得每个做模型的人放进雷达：

- **EU AI Act（通用型 AI／GPAI 义务）**：若你蒸馏后**对外发布**一个通用型模型，在欧盟法下你很可能就是「GPAI 提供者」，需承担技术文档、**著作权合规政策**、训练数据摘要揭露等义务；达到系统性风险门槛（以训练算力衡量）者义务更重。蒸馏不会让你「绕过」这些义务——反而可能让你**凭空继承一个提供者的全套责任**。
- **出口管制（美国 EAR 等）**：先进 AI **模型权重本身**，正逐步被纳入出口管制的讨论与规则框架。你蒸出并散布的权重，在某些情境下可能就是受管制的标的——这跟「软件开源就自由」是两回事。

> ⚠️ 真伪提醒
> EU AI Act 的 GPAI 细则、与美国对模型权重的出口管制框架，**都处于高速演变、甚至反复的阶段**（相关出口管制规则在 2025 年曾数度调整与重估）。具体门槛、生效时点、适用范围，请以**最新官方法规原文**为准；本书只负责标出「**这两道墙正在升起**」这个方向，不为任何具体数字背书。

## 情境 2　经济账：偷，可能比买还贵

**背景**：很多人以为蒸馏是「省钱」——不付昂贵 API 费，自己蒸一个放本地。但把第 1 章那道「污染之盾」的成本算进去，这笔帐未必划算。

**一笔被严重低估的帐**：

| 成本项 | 内容 | 为何被低估 |
|---|---|---|
| **采集成本** | 多帐号、代理 IP 池、API 费；帐号被封要重买 | 以为「采一次就够」，实则防护盾逼你反复采 |
| **除污成本** | 教师降转污染数据（第 1 章），采十万条能用的可能剩三万 | 还要再烧一轮算力 + 另一个可信模型判断哪条被污染 |
| **训练成本** | 全参数微调 35B 的 GPU 时数 | 一次调不好就要重来，迭代成本叠加 |
| **维护成本** | 教师一更新/下架，你的数据就过时，得重采重训 | 你的模型「保鲜期」绑在别人的发布节奏上 |
| **法律期望成本** | 被追究几率 × 后果 | 对企业而言，这项可能是**最大的一项**，却最难被算进 spreadsheet |

把这些加总，**和「干脆合法付费用官方 API」相比，蒸馏的省钱神话往往不成立**。这正是第 1 章那道盾最聪明之处——**它不阻止你，它只是让「偷」变得不划算。**

**把「合法蒸自己的」与「偷别人的闭源」并排，帐就一目了然。** 上面那张表算的是「偷」的隐性成本；下面这张，是把两条路放在同一把尺上量——你会发现省钱神话不只是「未必成立」，而是**在企业尺度上几乎必然倒挂**：

| 维度 | 合法路：蒸自有／开放权重 | 非法路：偷闭源 API |
|---|---|---|
| 采集合法性 | 清白，授权白纸黑字 | 违约，可能触刑 |
| 供给稳定性 | 权重在你手上，永不下架 | 绑死他人发布节奏，一改版／下架即断供 |
| 可追溯风险 | 无 | 学生可能**继承教师的统计指纹／水印**，事后可被验出 |
| 可商用／可审计 | 可商用、可过审计、可写进尽职调查 | 写不进任何合规文档，尽职调查一查就爆 |
| 监管义务 | 正常承担 GPAI 等义务即可 | 义务照样有，外加一笔违法事实 |
| 总成本曲线 | 前期投入高、之后摊平、可预期 | 看似前期省，后期被「持续采集＋持续除污＋法律期望成本」拖成无底洞 |

**「可追溯风险」这一列，值得单独拉警报。** 学术界早有「放射性数据（radioactive data）」的研究：训练数据会在模型权重里留下**可被统计检测的痕迹**；近年的文本浮水印（统计式 watermark 一类）与成员推断攻击，更把「这个学生是不是蒸了我的输出」从玄学变成**可举证的技术问题**。这意味着一件让人脊背发凉的事：偷来的能力**不会因为你关掉采集脚本就变干净**——它已经焊进了权重，成为对方未来任何一天都能拿出来的呈堂证据。**你以为偷完船过水无痕，实际上你亲手把指纹烤进了模型，而且烤进去就拿不出来了。**

> 💡 君之一席话
> 与君一席话：**最好的防盗，不是更高的墙，而是让偷窃的总成本高过购买——当「偷」比「买」还贵，诚实就成了最理性的选择，而非最高尚的选择。** 这是安全经济学最深的一课：你无法阻止所有攻击者，但你可以重塑他们的损益函数，让攻击在「期望值」上变成亏本生意。Anthropic 那道污染之盾的天才，不在于技术多难破，而在于它把蒸馏者的成本结构从「一次性采集」改写成「持续采集 + 持续除污 + 持续维护 + 持续承担法律风险」——四个「持续」叠加，再对比官方 API 那条清晰、合法、有 SLA 的定价，理性的人自己就会放弃。**真正成熟的防御，是让对手在计算机前算着算着，自己叹口气关掉了终端机。**

## 情境 3　六条铁律：让「想蒸馏」这件事安全落地

**背景**：把「想要本地强模型」这个完全正当的需求，导向安全合法的轨道。

1. **先问教师是谁**。蒸馏**开放权重模型**（Qwen、Llama、Mistral、GPT-OSS，遵守其授权）——绿灯。蒸馏**闭源商业 API**——红灯。这一条过滤掉九成麻烦。
2. **读懂授权，不只读懂 ToS**。开放权重 ≠ 随便用。有些禁商用、有些要求标注、有些限制衍生模型再授权。**用之前把 LICENSE 读完**（Llama 的可接受使用政策、Qwen 的授权、Apache-2.0/MIT 的差异都要分清）。
3. **数据集要查身世**。任何 `claude-traces`、`gpt-dump` 类数据集，使用前确认来源合法、授权清楚、上传者可信。**来路不明的原料，再香也别下锅。**
4. **绝不主动绕过技术防护**。速率限制、防护分类器、访问控制——这是厂商划的界。绕过它们，是把「违约」升级成「违法」的那一步，**永远不要踏出去**（见情境 1）。
5. **去自我宣称 ≠ 抹除来源**。在自有模型上做品牌中性化是正常工程；在他人模型数据上抹除来源痕迹，是湮灭证据。**同一技术，认清你站在哪一边。**
6. **企业场景，法务先行**。任何牵涉商业模型输出的训练计划，**先过法务、再写程序**。工程师的浪漫，不该变成公司的官司。

**第 1、2 条值得展开：开放权重不等于「随便用」，授权地景比想像中崎岖。** 把几种主流授权摆在一起，你就知道为什么「读 LICENSE」不是官腔，是保命：

| 授权类型（代表）| 商用 | 关键限制／义务 |
|---|---|---|
| Apache-2.0 / MIT（多数 Qwen、部分 Mistral）| ✅ | 最宽松；保留版权声明即可，Apache 另含专利授权 |
| Llama 社群授权（Meta Llama 系）| ✅ 附条件 | 自订授权＋可接受使用政策；超大用户量（MAU 门槛）需另谈授权；衍生模型有命名／标注要求 |
| Gemma 授权（Google）| ✅ 附条件 | 受 Gemma 条款＋禁用政策约束 |
| 部分厂商自订授权（某些大尺寸权重）| 视情况 | 可能含 MAU 门槛、再授权限制或特定领域禁用 |

> ⚠️ 真伪提醒
> 上表是**结构性导览**。各家授权的细节——尤其「能否用输出去改进其他模型」「MAU 门槛的具体数字」「衍生模型命名规则」——**版本间会变动**，本书成稿与你阅读之间可能又改过。**任何一次商用前，以该模型该版本的 LICENSE 原文为唯一依据**，别靠记忆、别靠本表。

**第 3 条的正面解法，是合成数据。** 与其去下载来路不明的 `claude-traces`，不如用**一个授权清楚的强开放模型**，搭配 `distilabel` 这类框架**自己生成**高品质合成 CoT 与指令数据——原料、过程、产出三段全在阳光下，可商用、可审计、可重现。这条路的品质正逐年逼近「偷来的轨迹」，而它最大的优势甚至不是品质，是**干净**：你能对任何人、任何审计、任何法庭，解释每一条数据从哪来。

**而把这六条铁律走到极致、最具商业价值的那个用法，叫「蒸馏你自己的模型」。** 企业手上往往已有一个又大又贵、跑在云端的内部大模型；把它的能力蒸进一个轻量学生，部署到边缘、端侧或本地内网——**延迟更低、成本砍掉一个数量级、数据不出私网、IP 100% 属于自己、过得了任何尽职调查**。这正是第 5 章那套 M2–M5 流水线最正当、最赚钱的落点：教师是你自己的模型，全程零红线。**注意这里的对称之美：同一套蒸馏技术，只要把教师换成「你有权榨取的那一个」，整件事就从『攻击』瞬间变回『工程』——分水岭从来不在代码，在你对那个教师，有没有授权。**

> 🔍 高端点评──合法的路不是「次优选择」，它正在变成「最优选择」
> 这六条铁律读起来像「约束」，但换个角度看，它们其实在指向一条越走越宽的阳关道，而这条道的回报正在快速超过那条红线。**第一，开放权重生态已经够强。** 2026 年的 Qwen、Llama、Mistral、GPT-OSS 系列，能力已强到配上第 4 章的蒸馏方法论 + 第 3 章的 DGX Spark，你完全能在零红线的前提下蒸出一个跑在自己机器上、可商用、可微调、能睡得安稳的强模型。**第二，合成数据正在崛起。** 用强开放模型 + `distilabel` 自动生成高品质合成 CoT，已是完全合法、且品质逐年逼近「偷来的轨迹」的路线——你蒸的是「开源模型的影子」，但这个影子本身合法、可控、且越来越聪明。**第三，蒸馏自有模型是企业刚需。** 把公司内部的大模型能力蒸给轻量模型部署到边缘/端侧，这是第 5 章那套 M2-M5 流水线最正当、最有商业价值的用法——没有任何红线。**所以真正成熟的姿态是：把追求『本地、私有、可控的强模型』这个理想，与『绕过别人的盾去偷』这个手段彻底解绑。** 理想完全正当，手段大有选择。绕过别人的盾，从来不是唯一的路，甚至——把帐算到最后——不是最划算的那条路。

## 情境 4　回到那个最初的问题

**背景**：全书始于一句搜索——「目前蒸馏 Anthropic 的 Fable、Mythos 模型的开源模型有哪些？」走到这里，可以给出一个远比清单诚实的回答。

- **如果你问的是「有哪些现成的」**：坊间流传有 Qwable-v1、Qwythos-9B、FableVibes 等（第 2 章）——但它们真伪难辨、品质受「污染之盾」拖累、来源多半踩 ToS 红线。**这份清单与其说是宝藏图，不如说是一张雷区图。**
- **如果你问的是「我该怎么拥有一个本地强模型」**：答案不在那份清单里，而在第 3 到第 6 章——**选一个合法的开放权重基底、用合法的数据、用成熟的蒸馏与量化方法论、跑在像 DGX Spark 这样的好机器上。** 这条路慢一点、朴素一点，但它是你真正能拥有、能商用、能睡得安稳的那一个。

> 💡 君之一席话
> 与君一席话：**人们以为自己想要的是「一个免费的 Claude」，其实真正想要的，是「一个只属于我、听我的、跑在我桌上的智能」——前者要靠偷，后者可以靠造，而只有靠造出来的东西，才真正属于你。** 偷来的影子，住在别人的法律阴影里，绑在别人的发布节奏上，藏着别人可能验出的水印；它越好用，你越睡不安稳。自己造的模型也许笨一点、慢一点，但它的每一个权重都来路清白，它的能力上限由你的数据与工夫决定，它不会在某个清晨因为一纸下架令或一封律师信而崩塌。**技术自由的真谛，从来不是「免费取得最强的能力」，而是「完全掌控一个够用的能力」——能掌控的平庸，胜过不能拥有的卓越。**

## 第一乐章小结：你已经有能力造出它了

技术会过时，Fable 5、Mythos、Qwythos——这些名字明年可能就没人记得。但第一乐章想留下的两个判断，会比任何模型活得更久：

1. **看懂蒸馏**：它是一门优雅的科学（软标签、KL 方向、CoT 对齐、轨迹反转、混合量化），值得每个工程师掌握；它的工具全部开源免费，真正稀缺的是「该保住什么、可牺牲什么」的判断力。
2. **看清那条线**：蒸馏你有权使用的模型，是工程；绕过防护去榨取别人的闭源模型，是攻击。**分水岭不在技术，在意图与授权。**

> ⚠️ 第一乐章免责
> 第一乐章为教育与研究用途，是对相关公开数据与技术现况的整理与批判性评注。书中所有对未授权能力提取的描述，**均为风险揭露，不构成操作建议**。愿你用这本书，造出一个真正属于你的智能——而不是去偷一个影子。

走到这里，第一乐章画上句点：你已经知道怎么在自己的桌上，合法地造出一个只属于你的本地模型。但**造出来，只是故事的起点**。

一个只会在对话框里回答问题的模型，再聪明，也还只是一个躺在硬盘里的大脑——它有思想，却没有手脚。而你蒸馏、量化、塞进那台 DGX Spark 的这个又小又便宜的模型，它真正惊人的价值，恰恰不在于「再多答对一道题」，而在于它够便宜、够快、够贴身、够私密，可以一刻不停地睁着眼、动着手——去**看懂一张屏幕、操作一台电脑**。昂贵的云端智能只配当顾问，廉价的本地智能才配当器官。

第二乐章，我们就让这个你亲手造出的智能，第一次真正地**动起来**。翻到第 7 章——看一个本地代理，如何学会「看懂」并「操作」一台为人类双眼设计的电脑。
