# 附录C　争议与认知问答

> 本附录整理蒸馏争议的内核问答：蒸馏为什么有争议、到底提取的是权重还是推理逻辑、为什么只学答案会失败，以及小模型如何在训练中长出「像会推理」的行为。本附录只做教育性整理；涉及闭源商业 API 的未授权蒸馏，仍以第 6 章红线为准。

## C.1　蒸馏的三层争议

有一句很准确的判断：知识蒸馏是「技术上公开，商业上敏感」的技术。争议不是因为 KD 本身邪恶，而是因为它同时踩到三组利益边界。

| 争议 | 表面问题 | 深层问题 | 工程判断 |
|---|---|---|---|
| **ToS 与授权** | 能不能用 API 输出训练竞品 | 合约禁止常比版权问题更直接 | 闭源商业 API 输出不可当免费训练料 |
| **搭便车与不公平竞争** | 小模型用低成本学到大模型能力 | 教师的算力、数据、安全成本被外部化 | 合法蒸馏应限于自有模型、开放权重或授权数据 |
| **安全能力被洗掉** | 学到能力，未必学到防护 | 学生可保留推理能力但弱化拒答与风险判断 | 评估不能只看分数，还要看安全与拒答品质 |

> 🔍 高端点评──「大家都在做」不是免责理由
> 大公司私底下做蒸馏，常见理由有三个：降低推理成本、把能力压到端侧、用强模型生成合成数据做评估。这三件事本身都合理；问题在于**教师是否授权、数据是否可追溯、用途是否与条款冲突**。工程上真正成熟的做法，不是把灰色地带说成产业惯例，而是把管线设计成可替换教师：同一套 M2-M5 流水线，教师可以是自家模型、开放权重模型、或有授权的商业模型。只要教师一换成「未授权闭源 API」，同一套技术就从 MLOps 变成风险事件。

## C.2　蒸馏提取的是权重，还是推理逻辑？

答案：**不是拷贝教师的物理权重，而是让学生用自己的权重重建教师的行为分布、偏好与推理轨迹。**

这点很重要。闭源教师的权重不可见；即使白盒可见，教师与学生的张量形状、层数、注意力头、词表也通常不同，没有「把 120B 权重塞进 9B」这种物理操作。蒸馏真正迁移的是三类信号：

1. **输出串行**：教师如何回答，学生学文本表面与格式。
2. **几率分布**：教师在多个可能 token 间如何犹豫，学生学「暗知识」。
3. **推理轨迹**：教师如何拆题、回推、使用工具，学生学问题解法。

用生活化比喻：提取权重像把老师的大脑切下来塞给学生，既不可能也没有意义；蒸馏像让老师大量示范解题，学生的大脑还是自己的，但解题习惯被重新塑形。

## C.3　为什么「答案是 42」不是好训练数据

常被提到的「答案是 42」这个梗，重点不在科幻典故，而在它对蒸馏的提醒：**没有问题框架与推理过程的答案，是低密度数据。**

如果训练集只有：

```text
问题：生命、宇宙与万物的答案是什么？
答案：42
```

学生只学到字符串对应，学不到为什么这个回答成立、何时只是玩笑、何时应该追问「你真正想问的是什么」。这正是硬标签 SFT 的天花板。

更好的蒸馏样本至少包含：

- 问题语境：用户是在问梗、数学、还是哲学？
- 推理路径：模型如何辨识典故与语用目的？
- 反事实边界：哪些情况下不该回答 42？
- 最终答案：短答、长答、或追问澄清？

> 💡 君之一席话
> 与君一席话：**答案是压缩档，推理是解压密码。** 只给学生答案，就像给他一个打不开的压缩包；他可以记住文件名，却无法重建内容。好的蒸馏不是让学生背更多「42」，而是让它学会判断什么时候 42 是玩笑、什么时候 42 是错答、什么时候问题本身需要被重写。

## C.4　大模型已经固化，为什么学生还能学到推理？

教师固化，指的是教师权重不再更新；这不代表教师不能产生新的训练信号。蒸馏时，教师像一台固定的高品质标注器：每次面对不同题目，都会产生不同的输出、不同的中间步骤、不同的工具调用路径。

学生学习发生在自己的梯度更新中：

1. 学生先根据目前权重产生预测。
2. 训练系统比较学生预测与教师信号。
3. 损失函数把差距转成梯度。
4. 优化器微调学生权重。
5. 重复数万到数百万次后，学生的行为分布向教师靠拢。

所以「推理写入」不是把教师内心搬过去，而是**用大量带结构的示范，改变学生在下一个 token 上的条件几率**。当这种改变跨过足够多任务、足够多中间步骤后，看起来就像学生学会了推理。

## C.5　小模型的学习过程：三个日常比喻

**学厨**：只看成品照片，是硬标签；看师傅每一步火候、试味、补盐，是 CoT 蒸馏。小厨师最后做出的菜不是师傅那锅，但味型会靠近。

**学开车**：只背「红灯停、绿灯行」，遇到施工、救护车、无号志路口就慌；坐在教练旁边听他说「先看左后视镜、再判断行人速度」，才是在学决策流程。

**学调试**：只给修好的代码，学生会背 patch；给错误消息、排查顺序、假设验证、回滚原因，学生才会学会 debug 的节奏。

## C.6　整理正文的判断

原始素材后半段把蒸馏讲得更激进，本书整理时做了三个取舍：

1. **保留机器学习原理**：权重 vs 逻辑、logits、CoT、学生如何学推理。
2. **保留工程可用性**：如何设计数据、基准、管线、工具选型。
3. **降级高风险内容**：对未授权蒸馏、去审查、越狱测试，只保留风险分类与防护观点，不转录操作步骤。

这样处理后，原始素材的价值不再是「一份可照做的灰色操作纪录」，而是一份可被工程师安全阅读的反面教材与方法论补充。
