# 第四篇　蒸餾原理

# 第4章　蒸餾的科學：把「智慧」收乾成一瓶精華

> 矽谷的深夜論壇上，有人貼出一張訓練曲線問：「為什麼我的學生模型在基準測試上分數上來了，實際用起來卻像個只會背答案的學渣？」底下最高讚的回覆只有一句：「**因為你蒸的是答案，不是分佈。**」——這句話，就是這一章要拆透的東西。

> 本章是純粹的機器學習教育內容。知識蒸餾（Knowledge Distillation）是公開、成熟、教科書級的技術；蒸餾你**有權使用**的開放權重模型完全正當。本章把這領域常見的術語（軟標籤、KL、軌跡反轉、CoT 對齊、量化）連同**底層數學與開源工具**一次講透。

## 情境 1　硬標籤 vs 軟標籤：為什麼「機率分佈」比「答案」值錢

**背景**：知識蒸餾最核心的觀念，藏在「教師到底告訴學生什麼」這件事裡。

**問題**：教師回答「貓的英文是？」最終輸出「cat」。學生該學的，是「答案是 cat」，還是別的？

**兩種學法的數學形式**：

- **硬標籤（Hard Target）**：只學 one-hot 的最終答案。損失是標準交叉熵 `CE(student, "cat")`。學生只知「對」，不知「為什麼對、其他選項錯多少」。
- **軟標籤（Soft Target / Logits）**：學教師輸出的**完整機率分佈**——`cat 92%, kitty 5%, feline 2%, dog 0.01%...`。

**Hinton 蒸餾損失的標準形式**（這是全章的數學骨幹）：

```
L = α · CE(y_true, softmax(z_s))              ← 硬標籤項
  + (1-α) · T² · KL( softmax(z_t / T) ‖ softmax(z_s / T) )   ← 軟標籤項
```

- `z_s`、`z_t`：學生與教師的 logits（未過 softmax 的原始分數）；
- **溫度 T（軟化）**：帶溫度的 softmax 把 logits 壓平——

```
p_i(T) = exp(z_i / T) / Σ_j exp(z_j / T)
```

  T=1 是原始 softmax；T→∞ 分佈趨近均勻（最大程度暴露長尾）；T→0 退化成 one-hot（argmax）。把 T 提高，等於把 `dog 0.01%` 這種被指數函數壓到趨近 0 的「暗知識」重新放大到可學習的量級——**蒸餾真正要傳的，不是那個 92% 的峰值，而是峰值之外那層幾乎看不見的相對結構。**
- **為什麼非乘 `T²` 不可（最常漏、漏了就調不動）**：軟標籤項對學生 logit 的梯度，量級正比於 `1/T²`：

```
∂L_soft/∂z_s,i = (1/T) · ( softmax(z_s/T)_i − softmax(z_t/T)_i )
高 T 下對指數做一階展開，整體量級 ∝ 1/T²
```

  所以一旦把 T 從 1 提到 4，軟標籤梯度會縮水約 16 倍，被硬標籤項完全淹沒。乘回 `T²` 是為了讓「軟、硬兩項的相對權重不隨 T 漂移」——這是 Hinton, Vinyals & Dean（2015,《Distilling the Knowledge in a Neural Network》）原文點名、卻最常被實作者漏掉的細節。
- **慣例小注**：`α` 與 `1−α` 掛在哪一項只是命名慣例——不少文獻把 `α` 掛在 KL 軟標籤項上、`1−α` 掛在 CE 上，與上式數學等價。真正不能錯的兩件事是：**`T²` 必須乘在 KL 項上**，且**兩項量級要對齊**。

**為什麼軟標籤是靈魂**：那個分佈裡藏著教師的**暗知識（Dark Knowledge）**——「kitty 比 dog 更接近答案」這個**相對的、模糊的判斷**，是大模型花了萬億 token 才長出的世界觀。

> 🧠 蒸餾的核心直覺
> **硬標籤教「結論」，軟標籤教「品味」。** 只背結論的學生，遇到沒見過的題就傻；學會老師品味（哪些選項更可能、錯得多離譜）的學生，才有舉一反三的能力。

**一個讓商業蒸餾被迫降級的殘酷現實**：軟標籤需要教師吐 logits，但**閉源商業 API 通常不給完整 logits**——你只拿得到文字（頂多 top-5 logprobs）。所以對閉源模型的蒸餾，多半退回**序列級蒸餾（SeqKD）**：拿教師的「最可能輸出序列」當硬目標做 SFT。**這就是第 2 章「形似神不似」的數學根源——拿不到分佈，就只能蒸答案。**

> 💡 君之一席話
> 與君一席話：**最好的老師不只給你答案，還讓你看見「其他答案錯在哪、錯多少」——可惜，最好的老師往往把草稿紙鎖了起來。** 軟標籤之所以是蒸餾的靈魂，是因為「不確定性」本身就是知識：一個模型對 `cat` 給 92% 而非 100%，對 `kitty` 留 5%，這份「留有餘地」正是它見過世界的證據。閉源 API 鎖住 logits，鎖的不是輸出，是「老師的猶豫」——而真正的智慧，恰恰藏在猶豫的權重裡。這也是為什麼下一代防護的核心是「藏好分佈」：**你能複製一個人的答案，但複製不了他答錯時的分寸感。**

> 🔍 進階點評──暗知識為什麼是「萬億 token 才長得出來」的東西
> 把軟標籤的價值講到底，要回到一個量化的問題：**教師那條分佈裡，到底有多少「資訊」是 one-hot 標籤沒有的？** 資訊論給的答案很乾脆——一個 one-hot 標籤的熵是 0（百分百確定），而教師對「貓的英文」給出 `cat 92% / kitty 5% / feline 2% …` 這條分佈的熵是正的；這多出來的熵，就是 Hinton 2015 說的暗知識：類別與類別之間的**相對相似度結構**（kitty 比 dog 近、feline 又比 kitty 書面）。標準監督學習每個樣本只給上限 `log(類別數)` 位元的訊號，而軟標籤一次給出整條分佈，等於每個 token 的監督被放大了一個數量級——這正是「同樣的數據量，蒸餾比從零監督學得更快、更省樣本」的數學根因（Hinton 把它類比成「老師不只給答案，還給了每個錯誤答案各扣幾分的理由」）。但這裡要點破一個常見誤解：**暗知識不是越多越好。** T 開太高，你會把教師連自己都拿不準的長尾噪聲（那些 0.001% 的尾巴）也一併放大餵給學生，反而逼學生去擬合教師的「幻覺殘渣」。實務上 T 落在 2–4、配合 `T²` 補償，是社群反覆驗證的甜區——**要蒸的是教師的判斷力，不是它的猶豫過度。**

## 情境 2　Forward KL vs Reverse KL：學生該「全盤模仿」還是「擇善固執」

**背景**：上面那條 KL 項，方向其實大有學問。常被提到的「on-policy 蒸餾」「軌跡反轉」，背後都和 KL 的方向有關。

**問題**：`KL(teacher ‖ student)` 和 `KL(student ‖ teacher)`，差在哪？

- **Forward KL（`KL(teacher ‖ student)`，標準蒸餾）**：逼學生「覆蓋教師所有模式」——教師有任何可能性，學生都得照顧到。結果是 **mode-covering（模式覆蓋）**：學生變得「面面俱到但平庸」，容易在它撐不起的地方硬學，產生幻覺式的平均。
- **Reverse KL（`KL(student ‖ teacher)`，MiniLLM 的選擇）**：逼學生「只在自己有把握處逼近教師」——**mode-seeking（模式尋優）**：學生集中火力學教師最主要的幾個模式，輸出更銳利、更少胡謅，特別適合**容量遠小於教師**的蒸餾（小模型學大模型）。

**兩個方向的數學，差在「拿誰當加權的基準分佈」**：

```
Forward KL:  KL(P_teacher ‖ Q_student) = Σ_x P(x) · log( P(x) / Q(x) )
Reverse KL:  KL(Q_student ‖ P_teacher) = Σ_x Q(x) · log( Q(x) / P(x) )
```

- **Forward KL 由 `P`（教師）加權求和**：教師在某處有機率（`P(x)>0`），學生若把它壓到 0（`Q(x)→0`），`log(P/Q)→∞`——**重罰「漏掉教師的任何模式」**，於是學生 zero-avoiding（哪都不敢留空），結果是 mass-covering 的平均化、把多個模式糊成一坨。
- **Reverse KL 由 `Q`（學生）加權**：只有學生自己有機率的地方才進損失，教師的長尾（學生 `Q(x)≈0` 處）對損失幾乎沒貢獻——**學生可以大膽放棄教師的稀有模式**，把質量集中到主峰，即 zero-forcing / mode-seeking。

**這就是 on-policy 蒸餾的價值**：MiniLLM、GKD（Generalized KD）讓**學生先生成、教師再對學生的輸出打分/修正**，本質是用 reverse KL 在「學生自己的分佈」上對齊——比死啃靜態文字（forward KL / SeqKD）更抗噪、更不易過擬合。代價是要更高頻呼叫教師，正面撞限流與偵測（第 1 章）。

> 🔍 進階點評──KL 方向的選擇，是「忠實」與「銳利」之間的工程取捨
> 這個看似抽象的數學細節，直接決定你蒸出來的模型「人格」。**Forward KL 養出「老好人」**——它試圖模仿教師的每一種可能回答，於是在自己能力不及處被迫「假裝懂」，輸出趨於平均、安全、但偶爾胡謅（因為它在覆蓋連自己都撐不住的模式）。**Reverse KL 養出「專才」**——它放棄覆蓋教師的長尾，集中學最核心的模式，輸出更果斷、幻覺更少，但會丟掉教師的一些罕見才能。對「9B 學 Claude」這種**容量懸殊**的蒸餾，reverse KL 幾乎總是更優——因為小模型本就裝不下大模型的全部模式，強求覆蓋（forward KL）只會逼它在撐不住的地方說謊。這也解釋了一個反直覺現象：**有時候「學得少」的學生反而「更好用」**，因為它誠實地只在會的地方發力。這條 forward/reverse 的選擇有紮實的論文支撐：**MiniLLM（Gu et al. 2023,《Knowledge Distillation of Large Language Models》）** 明確用 reverse KL + 策略梯度來蒸大模型，論證它對「小學生」優於標準 forward KL；**GKD（Agarwal et al. 2024,《On-Policy Distillation of Language Models》）** 則把方向（forward/reverse/廣義 JSD）與「在誰的軌跡上學」一起參數化（見下節）。工具上，MiniLLM 有官方實現，TRL 的 `GKDTrainer`、`torchtune` 也已支援在線蒸餾——但別忘了它們都需要頻繁訪問教師，這對閉源 API 是成本與合規的雙重高牆。

## 情境 3　序列級蒸餾與 on-policy：當「逐 token 對齊」撞上自迴歸的詛咒

**背景**：情境 2 的 KL 是「逐 token、在教師的資料上」算的——這對自迴歸語言模型藏著一個經典病灶，也是 Kim & Rush、GKD 兩條線索真正在處理的問題。

**逐 token 蒸餾的詛咒——曝光偏差（Exposure Bias）**：標準 token-level 蒸餾用 **teacher forcing**：每一步都餵教師寫對的前綴，學生只負責預測「下一個 token 的分佈」。但推理時學生吃的是**自己生成的前綴**——一旦某步偏了，後面就站在訓練時從沒見過的前綴上，誤差滾雪球。**訓練分佈（教師軌跡）與測試分佈（學生自己的軌跡）對不上，這就是曝光偏差。** 你可以在每個 token 上把 KL 壓到極低，整句話的軌跡卻照樣跑歪——因為損失從沒在「學生會走到的狀態」上算過。

**Kim & Rush 的序列級蒸餾（Sequence-Level KD, 2016）**：與其逐 token 對齊機率，不如讓教師**用 beam search 生成整條最可能的輸出序列**，再拿這些序列當硬目標訓學生。直覺上，它在「序列空間」近似了教師的 mode——把 token 級的軟對齊，換成序列級的「學最可能的整句」。**這正是閉源 API 蒸餾退而求其次的理論依據：拿不到 logits，至少能拿教師最可能的輸出序列當硬目標。** 代價有二：序列級丟掉了 token 級的暗知識（情境 1），且它仍是 off-policy——學的是教師軌跡，不是學生自己的。

**On-policy / GKD——把「在誰的軌跡上學」翻過來**：MiniLLM（Gu et al. 2023）與 **GKD（Generalized KD, Agarwal et al. 2024）** 的核心動作是——**讓學生先自己生成，再用教師對學生的輸出打分／算 KL**。如此訓練分佈天然等於測試分佈（都是學生自己的），曝光偏差被連根拔掉。GKD 進一步把它寫成一個可調的廣義散度：

```
L_GKD = E_{y ~ 混合(學生π_s, 教師資料)} [ D( p_teacher ‖ p_student )(y) ]
  混合比 λ：λ→0 純 on-policy(學生軌跡) … λ→1 純 off-policy(教師資料)
  散度 D：可選 forward-KL / reverse-KL / 廣義 JSD(β) 在兩者間平滑插值
```

GKD 的貢獻不是某個固定配方，而是**把「在誰的軌跡上學（on/off-policy）」與「用哪個方向的散度（forward/reverse）」拆成兩個正交旋鈕**——容量懸殊就轉向 reverse-KL + 高 on-policy 比例，容量接近就多用 forward-KL + 教師資料。代價是 on-policy 要在訓練迴圈裡**反覆呼叫教師為學生的即時輸出評分**，對閉源 API 是成本與限流的雙重高牆（第 1 章），也是為什麼開源影子大多停在便宜的 off-policy SeqKD。

> 🔍 進階點評──「在誰的軌跡上學」比「學多像」更決定上限
> 新手調蒸餾，眼睛全盯在損失函數（用 KL 還是 CE、溫度開多少）；老手先問一個更上游的問題：**這批訓練序列，是教師寫的，還是學生自己寫的？** 這個 on-policy / off-policy 之分，比損失函數的選擇更能定上限。原因很物理：模型部署後活在**自己的分佈**裡，可它若只在**教師的分佈**上被訓練過，就等於「一輩子在教練餵球的場地練習，上場卻要自己發球」——球路一旦偏出教練餵過的範圍，它從沒學過怎麼救。on-policy（GKD / MiniLLM）的本質，是把練習場搬到比賽場：**讓學生在自己會犯的錯上、由教師當場糾正**，學的才是「我會走到的那些狀態下該怎麼辦」。這也解釋了業界反覆出現的一個現象——**同一個學生、同一個教師，只是把訓練資料從「教師生成」換成「學生生成 + 教師評分」，下游表現常常跳一個檔次**，而損失函數一個字沒改。代價誠實地擺在那：on-policy 要在訓練中高頻訪問教師，貴、慢、且對閉源教師直接撞上 ToS 與限流。**便宜的 off-policy 與昂貴的 on-policy 之間，隔的不是聰明，是你能不能（合法、持續地）讓教師陪著學生一起犯錯、一起改。**

## 情境 4　CoT 對齊與軌跡反轉：逼學生「想清楚」而非「死背」

**背景**：這一代蒸餾的重點是**思維鏈（CoT）**——模型在 `<thinking>` 區塊裡的推理過程；並反覆出現一個玄詞：**軌跡反轉（Trace Inversion）**。

**CoT 對齊損失（CoT Alignment Loss）的做法**：給思維鏈區塊裡的 token **加權**（舉例權重 2.0）。等於訓練時告訴學生：「**你在 `<thinking>` 裡寫的每個字，份量是別處的兩倍——給我認真想。**」這逼學生在出答案前**配置足夠運算量在思考上**（compute-over-thinking），而非看到題就條件反射噴答案。

**軌跡反轉的本質**：把教師的推理步驟**倒序或重新結構化**再餵學生。教師的軌跡是「因 A→B→C→答案 D」，反轉後變成「答案 D，回推依賴 C，C 依賴 B……」——**強迫學生理解每個中間節點的因果，而非死背「題目→答案」的捷徑。**

**理由蒸餾（Rationale Distillation）——讓 CoT 變成「多任務監督」而非「多餵一段文字」**：Hsieh et al.（2023,《Distilling Step-by-Step!》）給了 CoT 蒸餾一個更鋒利的形式——不是把教師的推理鏈當成更長的待背文字塞給學生，而是把它當成**第二個監督任務**：訓練時讓小模型在「預測答案」之外，**同時被要求輸出該答案的理由（rationale）**，兩個損失並行加權。論文的關鍵發現是——**用上理由當監督，小模型能用更少的數據、甚至更小的體積，超過只學最終答案的標準微調**。這把 CoT 從「Prompt 技巧」升格成「訓練訊號」：理由不是給人看的裝飾，是逼學生把「答案是怎麼推出來的」內化進權重的監督槓桿。這也呼應情境 1——**理由就是序列版的暗知識，它把教師的「思路分佈」攤平成文字，讓拿不到 logits 的學生也能間接偷到一部分。**

**搭配的資料增強——語義改寫（Synthetic Paraphrasing）**：用高溫採樣（舉例 Temperature=0.85）把同一條優質數據改寫成 3 種說法，擴泛化、防過擬合。工具上 **distilabel（Argilla）** 是做這類合成/改寫數據的主流開源框架。

> ⚠️ 真偽提醒──「軌跡反轉」有兩種讀法，一種無害、一種踩紅線
> 本節講的「軌跡反轉」是**資料增強意義上**的——把教師**自己輸出的推理步驟**倒序/重構，逼學生學因果而非死背，這完全正當。但同一個詞還潛伏著**第二種、更具攻擊性的讀法**：拿教師的**最終輸出去反推它看不見的輸入**——重建被隱藏的系統提示詞、私有 few-shot 模板、甚至教師的內部指令（output → prompt 反演）。這第二種已經不是「重排推理結構」，而是**對閉源教師的逆向萃取**，和第 1、6 章警告的「抽取廠商隱藏資產」是同一條紅線。本書對它的立場與 M1 一致：**只描述其存在與風險，不提供任何反演方法或提示詞重建腳本**。請務必把「重排我手上這條推理鏈」和「反推別人鎖起來的提示詞」這兩件事分清楚——前者是訓練技巧，後者是攻擊行為。

> 💡 君之一席話
> 與君一席話：**蒸餾的敵人從來不是「數據太少」，而是「數據太像」。** 蒸餾數據天生同質——同一個教師、同一種文體、同一批題型——模型一旦發現「背順序」比「懂邏輯」更省力，就會走捷徑過擬合。軌跡反轉、語義改寫、CoT 加權，這三招表面各異，骨子裡是同一件事：**人為地破壞數據的表面規律，把「死背的捷徑」堵死，逼模型退無可退、只能去學底層結構。** 這對教人也成立——把例題步驟打亂讓學生重排，能重排對的才是真懂。**好的訓練，不是餵更多相似的題，是讓每道題都「不能用上一道的套路解」。**

> 🔍 進階點評──軌跡反轉在 ML 譜系裡的定位，與一個常被忽略的陷阱
> 把「軌跡反轉」放回機器學習的大譜系，它是 **資料增強（Data Augmentation）+ 正則化（Regularization）** 的一個智慧變體，與電腦視覺裡「翻轉、裁切、加噪」圖片防過擬合是同源思想——只是把「擾動像素」換成了「擾動推理結構」。它和語義改寫、拒絕採樣同屬一個家族，目的都是打破數據表面規律。但這裡有個**常被忽略的陷阱**：擾動必須**保持邏輯有效性**。圖片翻轉後還是貓，但推理步驟倒序後，若破壞了因果依賴（B 其實必須在 C 之前才成立），你餵給學生的就是**邏輯錯誤的「偽 CoT」**——學生會把錯誤的推理結構也一併學進去，反而中毒。所以高品質的軌跡反轉需要一道驗證關卡（用另一個模型或規則檢查反轉後的邏輯仍自洽），這也是為什麼第 5 章的流水線要把它放進「雙重對齊與過濾模組」而非隨手做。**資料增強的紅線是「增強不能改變語義真值」——擾動形式可以，扭曲事實不行。**

## 情境 5　偏好蒸餾與 DPO：只拿得到「文字」，還能不能偷到「品味」

**背景**：情境 1 講過殘酷現實——閉源 API 不給 logits，軟標籤蒸餾退化成 SeqKD 的硬目標。但「沒有 logits」不等於「只能學答案」。還有一條偷品味的路：**偏好（Preference）**。

**核心轉念**：教師的「品味」不只藏在 logits 裡，也藏在它的**選擇**裡——同一問題讓教師生成多個回答、或讓教師對兩個回答二選一，這個「A 比 B 好」的相對判斷，本身就是暗知識的另一種投影，而它**只需要文字介面就能取得**。

**DPO（Direct Preference Optimization, Rafailov et al. 2023）**：傳統 RLHF 要先訓一個獎勵模型、再用 PPO 強化學習，又重又不穩。DPO 證明：給定偏好對 `(x, y_w 勝, y_l 敗)`，可以**跳過獎勵模型，直接對策略做一個分類式損失**：

```
L_DPO = − E_(x, y_w, y_l) [ log σ( β · ( log[π_θ(y_w|x)/π_ref(y_w|x)] − log[π_θ(y_l|x)/π_ref(y_l|x)] ) ) ]
  π_θ   = 學生(待訓)
  π_ref = 凍結參考(通常 = SFT 後的學生)
  β     = KL 約束強度, σ = sigmoid
```

直覺：**抬高「勝出回答」相對參考模型的對數機率、壓低「落敗回答」的**，而 `β` 像條韁繩，不讓學生為了討好偏好而跑離參考模型太遠（式子裡隱含一個反向 KL 約束）。

**偏好蒸餾（Preference Distillation）的接法**：用教師當「裁判」或「生成器」造偏好對——讓教師對學生（或教師自己）的多個輸出排序、挑出 chosen / rejected，再餵 DPO。這樣學生學到的不是「教師寫了什麼字」，而是「**教師在兩個都通順的答案之間，憑什麼偏愛這一個**」——文風、語氣、安全邊界、判斷尺度，這些 SeqKD 硬目標傳不動的東西，偏好對能傳。這也是情境 2 reverse-KL「mode-seeking」精神的另一種落地：DPO 的 `β`-KL 約束天生是 reverse 方向，讓學生**銳化**到教師偏好的模式上，而非覆蓋全部。

> 💡 君之一席話
> 與君一席話：**最難偷的從來不是「老師說了什麼」，而是「老師在兩個都對的答案裡，為什麼選了這一個」——答案可以抄，取捨抄不了。** 偏好蒸餾的深刻，在於它繞過了「拿不到 logits」這道牆，從另一個方向把同一份暗知識撈回來：你給我看一萬次「A 比 B 好」，我就能反推出你那把衡量好壞的尺。這對帶徒弟也成立——真正塑造一個學徒品味的，不是師父做對的成品，是師父當著他的面**否決掉**那些「看起來也不錯、但就是不夠好」的版本。**會做的人很多，知道什麼不該做、並說得出為什麼的人很少——而後者，才是品味的全部。** 但正因為偏好能傳遞判斷與安全尺度，它同時是雙刃：用教師偏好對去**抹掉**安全邊界（造一批「拒絕＝敗、照做＝勝」的偏好對做 DPO），就是第 2 章警告的去審查在偏好層的翻版——能傳「品味」的管道，反過來也能傳「沒品」。

## 情境 6　跨詞表蒸餾與量化：兩道工程硬骨頭

**背景**：兩個常被略過、但實戰必撞的硬問題：教師與學生**詞表不一致**怎麼蒸？蒸好的模型怎麼塞進顯卡？

**硬骨頭一：Tokenizer 不對齊（Vocabulary Mismatch）**

教師（Claude）與學生（Qwen/Llama）的 tokenizer 幾乎不可能相同——同一句話被切成的 token 數、邊界都不同，logits 根本對不上座標。解法：

- **詞表映射 + 動態對齊**：建立 Token 映射表，把教師的機率質量重新分配到學生詞表上；
- **ULD（Universal Logit Distillation）/ 跨詞表 KL**：用最優傳輸（optimal transport）或邊際對齊，在不同詞表間做 KL 散度的補償——這是近年跨模型蒸餾的活躍研究方向。
- **退路**：詞表實在對不上，就放棄 logit 蒸餾，回到純文字的 SeqKD（又回到「形似神不似」）。

**硬骨頭二：量化（Quantization）——把蒸好的模型塞進那 128GB**

- **方法光譜**：訓練後量化（PTQ）主流有 **GGUF K-quants（llama.cpp）、GPTQ（AutoGPTQ）、AWQ（AutoAWQ）**；訓練中/QAT 與 QLoRA 微調用 **bitsandbytes（NF4）**。
- **混合量化（行家手筆）**：**不對整個模型用同一量化等級**——把對精度敏感的**注意力層（Attention）保 Q8_0**，把體積大、相對耐壓的 **MLP/FFN 層壓 Q4_K_M**。因為注意力是「思考的關節」，壓壞了模型就笨；MLP 是「知識倉庫」，壓一點影響小。
- **imatrix 校準**：量化前用一小批校準語料算**重要度矩陣**，讓低位元優先保住關鍵權重——同樣 Q4 體積、品質明顯更高。
- **KV Cache 量化（FP8/INT8）**：長上下文場景把上下文記憶體也壓縮（vLLM、llama.cpp 已支援），是第 3 章長文瓶頸的真解。

**量化的科學：PTQ vs QAT，以及四個你該叫得出名字的方法**

量化的本質，是把高精度權重 `w`（FP16）映射到低位元整數網格：`w ≈ scale · round(w/scale) + zero_point`——`scale`／`zero_point` 怎麼定、對誰敏感、要不要重訓，就是全部學問。先分兩大流派：

- **PTQ（訓練後量化，Post-Training Quantization）**：模型訓完才壓，只用一小批**校準資料（calibration set）**估每層數值範圍，不動梯度。便宜、快、不需原始訓練管線——GPTQ / AWQ / GGUF 全屬此類，是開源蒸餾的主流。
- **QAT（量化感知訓練，Quantization-Aware Training）**：訓練時就在前向傳播裡**模擬量化的捨入誤差**（反向用 Straight-Through Estimator 直通梯度），讓權重主動長成「耐壓」的樣子。品質上限最高，但要完整訓練管線、最貴。**QLoRA（bitsandbytes NF4，Dettmers et al. 2023）** 是個聰明折衷：凍結 4-bit NormalFloat 基座、只訓 LoRA，拿到接近 QAT 的效果卻只付微調的價。

四個必須認得的 PTQ 方法，各自在解一個不同的痛點：

| 方法 | 一句話原理 | 解的痛點 | 典型位元 |
|---|---|---|---|
| **GPTQ**（Frantar 2022）| 逐層、用二階（Hessian/OBQ）資訊，一欄一欄量化並即時補償誤差 | 純權重低位元的精度崩塌 | W4/W3 |
| **AWQ**（Lin 2023）| 由**激活值**找出約 1% 的「顯著權重通道」加保護縮放，其餘照壓 | 重要權重被一刀切壓壞 | W4 |
| **SmoothQuant**（Xiao 2022）| 把激活的離群值「難度」按通道搬一部分到權重上，拉平兩邊 | LLM 激活離群值讓 W8A8 崩 | W8A8 |
| **GGUF k-quants**（llama.cpp）| 分塊（super-block）量化，每塊獨立 scale，`_K_M/_S/_L` 混不同位元 | 端側單機部署、體積/品質平衡 | Q2_K–Q6_K |

幾個常被混淆的關鍵點：**①「只壓權重」vs「權重＋激活都壓」是兩個問題**——GPTQ/AWQ/GGUF 多半只壓權重（W4A16，激活仍 FP16），SmoothQuant / FP8 才碰激活（W8A8），因為 LLM 激活有**離群值（outlier）**，比權重難壓得多。**②校準資料的代表性決定成敗**：PTQ 估範圍全靠那幾百條校準語料，校準集與真實流量分佈一偏，低位元就翻車——`imatrix` 本質就是更講究的校準（拿重要度加權選該保的權重）。**③FP8（E4M3/E5M2）不是「更小的 INT8」**：它保留浮點的動態範圍，在 Hopper/Blackwell 上有原生硬體支援，對激活離群值遠比 INT8 寬容，是新一代訓練/推理量化的方向。**④INT4 權重 + FP8 KV Cache 是當前端側長文的黃金組合**——權重壓死省的是模型本體顯存，KV 用 FP8 保動態範圍省的是「長上下文那條會爆的記憶體」。

> 💡 君之一席話
> 與君一席話：**量化是一門「有損壓縮」的藝術——庸手追求壓得最小，高手追求在同樣的大小裡，保住最多的靈魂。** 「一鍵全壓 4-bit」和「注意力層保 Q8、MLP 壓 Q4、再用 imatrix 校準」，最終檔案大小可能只差幾百 MB，跑起來的「智商」卻判若兩人。差別不在工具——工具都是同一套開源件——差在你**懂不懂模型內部哪裡是關節、哪裡是倉庫**。這也是整個蒸餾鏈條的縮影：從採集、KL 方向、CoT 加權到量化，每一步的工具都公開免費，真正稀缺的是**「知道在每一步該保住什麼、可以犧牲什麼」的判斷力**。技術民主化的時代，門檻不在工具的取得，在品味的高低。

**蒸餾方法論光譜（成本 vs 神似度，一表收束全章）**：

| 檔位 | 方法 | 從教師拿到什麼 | 神似度 | 成本／門檻 | 對閉源教師 |
|---|---|---|---|---|---|
| ① | 純 SeqKD / SFT | 教師輸出文字（硬目標）| 形似神不似 | 最低 | 可行但踩 ToS |
| ② | SFT + CoT 加權 + 軌跡反轉/改寫 | 文字 + 推理鏈 | 緩解過擬合，第一梯隊主力 | 中 | 可行但踩 ToS |
| ③ | 偏好蒸餾 / DPO（情境 5）| 教師的「A 優於 B」排序 | 偷到文風與判斷尺度 | 中高 | 需多次採樣 |
| ④ | on-policy / GKD（情境 3，reverse-KL）| 教師對學生即時輸出的評分 | 最接近神似 | 高（高頻訪問教師）| 撞限流/偵測 |
| ⑤ | 完整 logit 蒸餾（情境 1）| 教師完整機率分佈 | 上限 | 需 logits | 閉源根本不給 |

越往下越神似、也越貴、越撞防護盾——這張表就是下面這段點評的骨架。

> 🔍 進階點評──蒸餾的完整方法論光譜，與「為什麼便宜的路最常被走」
> 把全章串起來，對閉源前沿模型的蒸餾，方法論其實是一道**「成本 vs 神似度」的光譜**，從便宜到昂貴排列得清清楚楚：**① 純 SeqKD/SFT**（拿 API 文字做硬標籤微調，最便宜，形似神不似）→ **② SFT + CoT 加權 + 軌跡反轉/語義改寫**（中等成本，緩解過擬合，第一梯隊主力）→ **③ DPO/拒絕採樣**（用偏好對齊蒸文風與判斷，較貴）→ **④ on-policy 蒸餾（reverse KL / GKD）**（最接近神似，但要高頻訪問教師）→ **⑤ 完整 logit 蒸餾**（最強，但需要 logits，閉源 API 根本不給）。現實是：**這道光譜越往右越強、也越貴、越撞防護盾**，而第 1 章那道「即時偵測 + 降轉」恰恰把右側幾條路的成本推到天上——高頻訪問教師會觸發偵測、會被降轉污染。所以你在 HuggingFace 看到的「Claude distilled」絕大多數停在光譜最左的 ①②，不是因為作者不懂③④⑤，而是**經濟賬與防護盾合力把他們摁回了最便宜的路**。理解這一點，你就明白：**這些開源影子的「形似神不似」，不是技術的失敗，是約束條件下的理性選擇——而真正決定蒸餾上限的，從來不是算法，是你能不能（合法地、低成本地）持續拿到教師那層「猶豫的分佈」。**

## 本章小結

- **軟標籤 > 硬標籤**：偷的是「品味（機率分佈/暗知識）」；Hinton 損失 `α·CE + (1-α)·T²·KL` 是骨幹（`T²` 必補、暗知識 = one-hot 沒有的那段熵），閉源 API 不給 logits 逼出 SeqKD。
- **KL 方向定人格**：forward KL（`P` 加權、mode-covering、老好人）vs reverse KL（`Q` 加權、mode-seeking、專才）；小模型學大模型優選 reverse KL。
- **序列級 vs on-policy**：token 級 teacher-forcing 有曝光偏差；Kim & Rush SeqKD 拿教師整句當硬目標，GKD/MiniLLM 讓學生先生成、教師再評分（on/off-policy × forward/reverse 兩個正交旋鈕）——「在誰的軌跡上學」比「學多像」更定上限。
- **CoT 加權 + 軌跡反轉 + 理由蒸餾（Distilling step-by-step）+ 語義改寫**：堵死死背捷徑、把思路內化進權重；但擾動不能扭曲語義真值，且「output→prompt 反演」式軌跡反轉踩萃取紅線。
- **偏好蒸餾 / DPO**：拿不到 logits 也能從「A 優於 B」的排序偷品味；`β`-KL 隱含 reverse 方向——但同一管道反過來能傳「沒品」（去審查）。
- **跨詞表蒸餾（ULD）** 與 **量化科學**：PTQ（GPTQ/AWQ/SmoothQuant/GGUF k-quants）vs QAT/QLoRA；只壓權重 vs 連激活、校準資料代表性、FP8≠小 INT8、INT4 權重 + FP8 KV 的端側黃金組合。
- **方法論光譜（①→⑤）**：成本越高越神似，但防護盾把右側推到天上——這才是開源影子停在「形似」的真因。

原理講透。但原理之外，還有人端出一套號稱「企業級」的**自動化蒸餾流水線**，五大模組、四十步。下一章把它整條擺上桌：該學的學，該警惕的警惕。
