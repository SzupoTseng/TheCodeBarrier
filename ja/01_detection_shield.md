# 第一部　反蒸留防衛線

# 第1章　検知の盾：なぜこの世代はとりわけ蒸留しにくいのか

> シリコンバレー、マウンテンビュー、深夜一時。とあるガレージで、一本の Python スクリプトが Fable 5 の API に三百問のプログラミング課題を立て続けに投げていた。最初の二百問、応答は教科書のように見事だった。だが二百一問目から、品質は目に見えて崩れ落ちた——同じ問いに対し、答えが突然短く、浅く、まるで別人が答えているかのようになった。ログを眺め続けてようやく気づく——**向こうは本当に「別人」に入れ替わっていたのだ。**

## 場面1　見えないのに、ずっと動き続けている盾

**背景**：直感に反する結論がある——**Claude Fable 5** をベースに蒸留されたオープンソースモデルは「**数が極めて少なく、しかも極めて初期段階にある**」というのだ。これは過去二年間の「新モデルが出れば数週間以内に distilled 版が氾濫する」という筋書きとは真逆である。

**問い**：なぜ今回は枯渇したのか。主な理由は二つあり、その技術的な核心を分解することは、結論を暗記するよりはるかに重要だ。

1. **組み込み型の蒸留検知（distillation detection）**：Fable 5 は史上初めて、防衛線を**推論経路そのものに組み込んだ**とされる——システムがユーザーによる**能力抽出（capability extraction）**の試みと判定するや、対話を自動的に旧世代の Opus 4.8 へ**ルート変更（reroute）**して処理させる、というのだ。
2. **輸出規制による撤去**：米政府の輸出規制指令により、Fable 5 は**公開発表のわずか数日後に全世界で利用停止**となったとされ、コミュニティが確保できた窓は約4日間、そこから削り取れたのは**わずか 4,659 件のクリーンな Agent 実行トレース**だけだったという。

> ⚠️ 真偽メモ
> 「Fable 5」は実在のモデルである。だが「リアルタイム蒸留検知＋ Opus 4.8 へのルート変更」「4日で輸出規制により撤去」「4,659 件のトレース」といった具体的な言説は、**本書では独自に裏付けが取れない**。コミュニティの噂、あるいは伝言ゲームの寄せ集めである可能性が高い。「都市伝説」として読むこと。Opus 4.8 を「格下の劣化品」と描く語り口そのものが怪しい——現実の Opus 4.8 は Anthropic の現行主力の一つである。

**この盾は技術的にどう機能しうるのか**：なぜこれが厄介なのかを理解するには、まず守備側がどんな武器を持っているかを見る必要がある。「リアルタイム蒸留検知＋ルート変更」システムは、技術的には三層から組み立てられる。

- **行動分類器（behavioral classifier）**：リクエスト側で軽量モデルを走らせ、「プロンプト分布」をリアルタイムで採点する。蒸留採集のトラフィックには強烈な指紋がある——**高度に構造化され、完全な思考連鎖を要求し、意味的多様性が低く、単一アカウントから高並列で、テンプレート化されたシステムプロンプト**。これはまさに学術的な「メンバーシップ推論攻撃（Membership Inference）」と「モデル抽出検知」を工学化したものだ。
- **出力側ウォーターマーク（output watermarking）**：デコード時に logits へ、鍵の保持者だけが検証できるバイアスをかける（緑/赤トークンのグループ分け、Kirchenbauer らの方式）。これにより「その出力で学習した生徒モデル」が、検出可能な統計的刻印を帯びるようにする。**君が蒸留したモデルは、本人すら気づかない場所に教師の指紋を残すのだ。**
- **拒否ではなくルート変更（reroute, not refuse）**：これが最も陰険な一手だ。採集の疑いを検知しても、エラーも返さず、アカウントも凍結しない——**ただ黙って、以降の応答を能力の劣るモデルに引き継がせる**。攻撃者が手にするデータは「正しそうに見えて、実は水増しされている」。

**蒸留者にとっての致命傷**：従来のアカウント凍結は**離散的で観測可能**だ——ブロックされれば、君にはそれがわかる。だがルート変更は**連続的で不可視**だ——十万件採集して、そのうちどの三万件が汚染されたか、わからない。育てた生徒モデルはどこを取っても中途半端な味がするが、君は自分の腕が未熟なせいだと思い込んでしまう。

> 💡 ひとこと助言
> ひとこと言わせてもらえば——**セキュリティの最高境地とは、対抗コストを「阻止」から「汚染」へ移すことだ。** 阻止は二値的で、観測可能で、攻撃者が即座に調整できる。汚染は連続的で、不可視で、しかも攻撃者の「自分のデータへの信頼」を内側から腐らせる。ブロックされた泥棒は鍵を替えてまた試す。だが贋物を盗んだことに気づかない泥棒は、その贋物を自らの手で祭壇に供えてしまう——捕まえるよりも徹底的だ。この原理を拡大すれば、情報セキュリティ、不正対策、コンテンツ審査のあらゆる場面でそれが見えてくる。**最強の防衛線は、君が負けたことを君に知らせない。**

> 🔍 掘り下げ解説──この盾の工学的現実と亀裂
> リアルタイム検知＋ルート変更の設計は極めて賢いが、現実には三つの亀裂があり、そのそれぞれが攻防進化の一方向に対応する。**第一、誤検知（false positive）。** 行動分類器は「プロンプトが採集っぽいかどうか」で採点するが、モデルに「show your full reasoning」と要求するのが大好きな正常なヘビーユーザーは、その特徴が蒸留者と高度に重なる——彼をルート変更すれば、体感は「今日の Claude、なんで急にバカになった?」だ。感度と誤検知率は永遠にシーソーである。**第二、ウォーターマークの脆弱性。** 出力ウォーターマークが恐れるのは二つ——言い換え（paraphrase）と集約だ。攻撃側は採集後に別のモデルで応答を**意味的に書き換える**だけで（第4章の synthetic paraphrasing 参照）、統計的刻印の大半が希釈される。学術的にもウォーターマーク除去攻撃（Krishna らの paraphrase attack など）はかなり成熟している。**第三、ルート変更の探知可能性。** 攻撃側は逆に「劣化検知器」を作れる——同じ問題を繰り返し問い、意味的類似度で応答品質が急落していないか監視し、ルート変更されたと気づいたら鍵を替えて再試行する。これはまさに第5章のパイプラインで M1 が内包するモジュールの一つだ。だからこの戦いの真相はこうだ——**盾は採集を阻止しない。ただ採集の単位コストを引き上げ、データの歩留まりを押し下げるだけだ**——歩留まりが「十万採って三万しか使えず、しかも汚染除去にもう一巡の演算力を焚く」ところまで落ちれば、蒸留の採算は崩壊する。盾の真の殺傷力は、城壁の上ではなく、帳簿の上にある。

## 場面2　なぜ DeepSeek のあの手口は、今回は効かないのか

**背景**：噂によれば、過去 Sonnet、Opus の世代は「大規模な悪意ある蒸留攻撃」を受けたことがあり、「DeepSeek やアリババが過去に告発された行為」が名指しされている。

> ⚠️ 真偽メモ
> 「Anthropic がアリババによる Claude 蒸留を告発した」というのは、確かに公開報道に存在する項目だ（付録A の出典リスト参照）。具体的な告発内容、それが成立するか、法的進展については、本書は判断を下さず、「業界で広く報道された蒸留をめぐる紛争が過去に起きた」とだけ記録する。

**問い**：以前は Sonnet／Opus に手を下せたのに、なぜ Fable 5 相手では無理なのか。鍵は、**過去の蒸留方法論が、この新しい盾と真っ向から相剋する**点にある。

**過去のあの手口——系列レベル知識蒸留（Sequence-level KD / SeqKD）**：商用 API は**logits を渡さず、テキストしか返さない**ため、攻撃側はソフトラベルを手に入れられず、次善の策を取るしかない。

- 教師 API を大量に呼び出し、「プロンプト→完全な応答テキスト」をデータセットとして収集する。
- このテキスト群を教師信号として生徒に指示チューニング（SFT）を行う——本質は **Kim & Rush の SeqKD** だ：教師の「最も確からしい出力系列」をハードターゲットとして使う。
- ツールチェーンは成熟している：`datasets` でトレースを保存、`trl` の `SFTTrainer` あるいは **Axolotl／LLaMA-Factory** でチューニング、`peft` で LoRA を載せる。**すべて既製のオープンソース部品だ。だからこそ DeepSeek 流の「力こそパワー」式採集が成立した——量が十分大きく、十分分散していれば、結局は十分な良質データを削り取れる。**

**なぜ今回は相剋するのか**：SeqKD の急所は、**「採集できたテキストの品質」に完全に依存する**ことにある。過去の反蒸留は**外周防衛線**（レート制限、アカウントのリスク管理、異常トラフィック検知）であり、これらはすべて工学で迂回できた——複数アカウント、プロキシ IP、リクエストの分散だ。だがこの新しい盾は戦場を**境界から一文一文の内側へと移した**。

- レート制限は迂回できても、ルート変更は迂回できない——ルート変更は**モデルが「君にちゃんと答えるかどうか」を決めるその瞬間**に起きる。
- どれだけ大量に採集しても、**源泉そのものが水増しされている**。SeqKD に食わせる「教師の最も確からしい出力」自体が格下品なのだ。

これはまさに業界の観察と呼応する——現存するオープンソース蒸留モデルは「大半が一般的な対話しか拾えず、かつての Sonnet／Opus のように大規模に蒸留することはできない」。

> 💡 ひとこと助言
> ひとこと言わせてもらえば——**防衛線を境界から核心へ内化させることは、セキュリティ工学において最も高価で、最も徹底した一手だ。** 境界防御（WAF、レート制限、リスク管理）の宿命は「迂回可能」であること。攻防双方がその壁を見ているからで、勝負は誰の工学がより勤勉かに帰着する。だが核心内化防御（検証・汚染・ウォーターマークを業務ロジックそのものに織り込む）は、「境界を迂回する」という動作そのものを完全に無意味化する——迂回すべき境界がもはや存在しないからだ。その代償は、セキュリティコストを正常サービス一回ごとのレイテンシと演算力に薄く溶かし込むこと。その見返りは、攻撃者が「壁の外で作業する」近道を二度と見つけられなくなることだ。**高価だが、一度やれば永続する——これこそ成熟したシステムが進んで払う対価である。**

> 🔍 掘り下げ解説──攻防の次の一回戦は「汚染除去」と「on-policy」にある
> この盾が本当に存在するなら、攻撃方法論の進化はほぼ運命づけられており、しかも方向は極めて具体的だ。**その一、「大量採集」から「品質校正」へ**：採集後に一工程を加え、別の信頼できるモデル（あるいは一貫性投票）で各トレースがルート変更で汚染されていないか判定し、水増し分を濾し取る——これが M1 の「劣化検知器」の由来であり、代償は歩留まりとコストの二重打撃だ。**その二、off-policy から on-policy 蒸留へ**：学術的に MiniLLM（reverse KL）、GKD（Generalized KD）は、生徒に**まず生成させ、それを教師が採点／修正する**オンライン蒸留が、静的なテキストを丸暗記するよりノイズに強く、過学習しにくいことを示した——だがこれは教師をより高頻度で呼び出す必要があり、レート制限と検知に真っ向から衝突し、コストはさらに高い。**その三、教師を迂回し合成データへ**：本物の教師が採りにくいなら、いっそ既存のオープンモデル＋ルールで合成思考連鎖を生成する（`distilabel`、self-instruct の類）。この道は完全に合法だが、君が蒸留するのは「オープンソースモデルの影」であって、Claude ではない。三つの道にはそれぞれ取捨があるが、共通して一つの結論を指す——**この盾は蒸留を殺したのではない。蒸留を「タダ飯」から「コストを計算しなければならない商売」へと引き戻しただけだ。**

## 場面3　検知エンジンの内臓：四種の信号と、逃れられない一枚の指紋表

**背景**：前二節では「リアルタイム検知」を一つのブラックボックスとして語ってきた。今度はそれを分解する——成熟した抽出検知システムは、**決して単一の必殺技に頼らない**。むしろ複数の**それぞれ単独では信頼できない弱い信号**を重み付け融合（ensemble）し、信頼できる判断に近づける。以下の四種の信号は、いずれも学術的に名前があり、論文があり、既知の対抗手段もある。攻防はこの四本の線の上で繰り返し綱引きされる。

**信号一　行動・トラフィック指紋（Behavioral / Traffic Fingerprinting）**——最も安価で、リクエスト側で走る。蒸留採集のトラフィックは統計的に生臭さを帯びる：**単一アカウントの高並列、リクエスト間隔が過度に規則的（機械のリズム）、プロンプトの意味エントロピーが低い（テンプレート化）、システムプロンプトが高度に酷似、判で押したように完全な思考連鎖を要求**。これらの特徴を軽量な異常検知器（孤立森林／一クラス SVM、あるいは単純なルール閾値）に食わせれば、内容の意味を見なくても「採集の疑い」スコアを先に付けられる。これは**レート制限とアカウントリスク管理の進化版**であり、攻撃側が「複数アカウント＋プロキシ IP ＋リズムの分散」で最も容易に迂回できる層でもある。

**信号二　出力ウォーターマーク（Output Watermarking）**——証拠をトークン一つひとつに織り込む。Kirchenbauer ら（2023）の**グリーンリスト（green-list）方式**を例に取れば、機構は手計算できるほど精密だ：

```
デコードの各ステップ：
  seed = hash(直前のトークン) ⊕ 秘密鍵
  seed を使って語彙表を疑似ランダムに「グリーンリスト(割合 γ)」と「レッドリスト」に切り分ける
  グリーンリストのトークンの logit にバイアス δ を加える          ← ウォーターマークはここで埋め込まれる
  通常どおりサンプリング（緑トークンの確率がこっそり持ち上がるが、文章は依然として自然）

検証（秘密鍵のみ必要、モデルは不要）：
  T 個のトークンの一段中、緑トークンの出現数 |s|_G を数える
  z = (|s|_G − γT) / sqrt(T·γ·(1−γ))
  z > 4  →  この文章がウォーターマークなしで自然発生する確率はほぼゼロ（誤報率 < 1e-4）
```

肝心な性質は三つ：**（1）検証側はモデルの重みを必要とせず、秘密鍵さえ持っていればオフラインで判定できる**；（2）ウォーターマークは統計的であり、単文では判定が難しいが長文では必ず現れる；（3）その天敵は**言い換え**だ——Krishna らの DIPPER paraphrase attack、および Sadasivan らの「AI 生成テキストは信頼できる形で検出できるか」という論文は、いずれも**重度の意味的言い換えが z 値を閾値以下へ希釈しうる**ことを示した。もう一つの技術路線は、Aaronson が提唱し OpenAI が採用したとされる **Gumbel／暗号学的サンプリングウォーターマーク**だ。理論上は「無歪み（distortion-free）」で出力分布を変えず、より気づかれにくいが、同じく集約的な言い換えには弱い。**君が蒸留した生徒モデルは、本人すら気づかない片隅で、教師のグリーンリストバイアスを吐き続ける——各トレースを先に一巡の演算力で書き換え、洗い清めない限りは。**

**信号三　カナリア文字列とハニートークン事実（Canary Tokens / Honeytokens）**——ウォーターマークの離散版であり、最も古い一手でもある。応答の中に意図的に**極めて稀な唯一の文字列**を植え込むか、事実回答の中に**この教師だけが言う架空の事実**を埋める（地図製作者は海賊版対策として地図に実在しない「罠の通り」を描き込み、辞書業者は架空の見出し語を仕込む。原理はまったく同じだ）。ある下流の生徒モデルが**このカナリアを復唱**したなら、蒸留関係は確定する——これは著作権およびデータ漏洩検知において法廷級の証拠であり、統計的ウォーターマークよりも堅い。

**信号四　メンバーシップ推論と分布指紋（Membership Inference & Logit Fingerprinting）**——事後に現行犯を押さえる。教師の出力で育った生徒は、**教師特有の logit 幾何とパープレキシティ（perplexity）／文体（stylometry）特徴を継承する**。候補の生徒モデルへの API アクセス権を得たら、メンバーシップ推論攻撃（Shokri らの shadow-model 法、Carlini らの **LiRA**）で「この生徒は私の出力で訓練されたか」を検証できる——頼みの綱はまさに「モデルは訓練済みのサンプルに対して損失が異常に低い」という綻びだ。**これは信号一から三を『採集の阻止』から『盗用の逆指名』へと格上げし、法律の盾の弾薬源となる。**

| ⚠️ メンバーシップ推論(LiRA) | 訓練済みサンプルの損失が異常に低い | Shokri 2017 / Carlini LiRA | 加噪、正則化、early stop | 「お前は俺を蒸留した」と逆指名される |

> 💡 ひとこと助言
> ひとこと言わせてもらえば——**検知工学の秘密のすべては、それぞれ単独では信頼できない弱い信号の束を、重み付けして信頼できる一つの判断にすることだ。** 決意ある相手を、単一の信号だけで止められるものは一つもない——ウォーターマークは書き換えられ、トラフィックは分散され、カナリアは希釈され、メンバーシップ推論は正則化で塗り潰される。だが四種の弱い信号を積み重ね、攻撃側に**同時に**すべてを迂回するよう要求すれば、必要な工学量と演算力は掛け算で爆発する。これこそ情報セキュリティの「多層防御（defense in depth）」の数学的本質だ——**乗り越え不可能な壁を一枚造るのではなく、壁を越えるという行為の総コストを、壁の向こうにある物の価値より高くすることだ。** 「一撃必殺」を期待するセキュリティ設計は、最後は必ず一撃で破られる。七割の信頼性しかない遮断を十個積んで九分九厘にしたものこそ、本当に手強い。この原理を肝に銘じれば、ファイアウォールを見ても、リスク管理を見ても、コンテンツ審査を見ても、それらがますます同じ姿に見えてくるはずだ。

> 🔍 掘り下げ解説──能動的ウォーターマーク vs 事後検知、そしてあの「不可能定理」
> この四種の信号の中には、攻防の天秤を決定づける**根本的な非対称性**が潜んでおり、単独で指摘する価値がある。学術的に Sadasivan ら（2023）は、ほとんど残酷とも言える結論を示した——**生成モデルの出力分布が人間に近づくほど、いかなる「事後検知器（post-hoc detector）」の精度上限も、出力分布と人間分布の全変動距離（TV distance）に縛り付けられる**——つまり、モデルが強くなるほど、テキストだけを見て「これは AI が書いたものか」を当てる道は必ず無効化へ向かう。これは守備側に悲観的に見えるが、肝心の転換点はここだ——**能動的ウォーターマーク（信号二）は事後に推測するのではなく、デコードのその瞬間に証拠を『植える』**。これはあの不可能定理を回避する——バイアスは既知の秘密鍵によって意図的に注入されたものであり、分布の差で推測しているのではないからだ。すると真の戦略図はこうなる——**『デコードの瞬間』を握る者が、検知権を握る。** API 提供者はデコード権を握り、ウォーターマークを植えられ、リアルタイムでルート変更もできる。第三者の蒸留者はテキストしか手に入らず、必敗の事後検知しかできない。この非対称こそ、なぜこの盾が守備側に偏るのかを説明する——**守備側がより賢いからではない。守備側が確率分布を書き換えられる側に立っているからだ。** そして攻撃側の唯一の打開策は、場面2 の古い問題に立ち戻る——各トレースを書き換えるために追加の演算力を費やし、植えられた証拠を洗い流すことだ——これまた、蒸留を「タダ飯」から「赤字商売」へ押しやる一筆のコストである。

## 場面4　城壁の外のもう一つの盾、法律：輸出規制、ToS、そして「蒸留犯罪化」の虚実

**背景**：技術の盾の外には、より緩慢で、より重く、それでいてエンジニアにしばしば見過ごされるもう一つの盾がある——**法律とコンプライアンス**だ。巷では、Fable 5 が「米政府の輸出規制指令により、公開発表の数日後に全世界で利用停止となった」という言説が流れている。この一文は、性質のまったく異なる三つの法律概念を一塊に混ぜ込んでいる。まず本物の法的地形を整理し、それから改めてこの言説の虚実を判断しよう。

**まず三本の独立した赤い線を区別する。その結末は天と地ほど違う**：

| 赤い線 | 法的性質 | 引き金 | 結末の規模 |
|---|---|---|---|
| サービス規約（ToS）違反 | 民事 / 契約 | 自動化採集、レート制限の迂回 | アカウント凍結、損害賠償請求、利用禁止 |
| 《コンピュータ詐欺・濫用法》(CFAA) 違反 | 米連邦刑事 | 「無許可アクセス」をめぐる争い | 認定次第で刑事責任に及びうる |
| 輸出規制（EAR/BIS）違反 | 米連邦刑事 | 規制対象品目を制限対象へ輸出 | 重罪、巨額の罰金 |

**輸出規制は結局何を規制するのか**：米国の **EAR（輸出管理規則）は商務省 BIS が執行する**。2022年10月と2023年10月の「先進演算」規則が**主に規制するのは演算ハードウェア**（A100／H100 等の高性能 GPU および製造設備）であり、特定国への輸出を制限する——規制対象は**チップであって、モデルの重みそのものではない**。モデルの重みが「規制対象品目」になるかどうか、どんな閾値でなるのかは、**いまなお進化中で、決着のついていない**最前線だ：大統領令 EO 14110（2023年10月）は「二重用途基盤モデル」に対し、訓練演算量の**通報閾値（約 1e26 FLOPs 級）**を設けた。2025年初頭、BIS は一時「AI 拡散（AI diffusion）」枠組みを打ち出し、最強の閉源モデルの重みの輸出までも規制対象に取り込もうとした——だがこれらは通報と輸出の規制であり、**中核ロジックは「特定の対象に物を渡してはならない」であって、「ある製品を世界中で四日以内に撤去する」ではない**。

**だからあの「輸出規制で即時全世界撤去」という言説の最も怪しい点**：輸出規制の道具は**移転対象の制限**（どの国、どの国籍の人が手に入れてはならないか）であり、それは「リコール令」ではない。あるモデルが本当に発表数日後に消えたのなら、**より合理的な説明は、コンプライアンス審査や内部評価のためのメーカーによる自主停止（voluntary pause）**であって、一枚の輸出規制令で「全世界利用停止」になったのではない。この二つを混同することこそ、都市伝説によくある継ぎ接ぎの痕跡だ。

> ⚠️ 真偽メモ
> 「Fable 5 が輸出規制により数日で全世界利用停止」「残された窓は4日だけ、削り取れたのは 4,659 件のトレース」というこの一連の言説は、**『輸出規制』『製品撤去』『窓の期間の数字』という、本来無関係な三つの事柄を一つの劇的な物語に縫い合わせている**。本書では独自に裏付けが取れず、コミュニティの噂か伝言ゲームの寄せ集めである可能性が高い。「都市伝説」として読むこと。加えて、前述の言説（および第5章）は「米国のテック界が大規模蒸留攻撃を**刑事犯罪化**する動きを推し進めている」と述べているが——現状、**『モデル蒸留』そのものを犯罪とする専用の法律は一切存在しない**。現実に適用されうるのは ToS（契約）、CFAA（無許可アクセス。しかもその適用範囲自体が司法上いまだ争いがある）、営業秘密、輸出規制などの**既存**枠組みの延長であって、新たな罪名ではない。「誰かが提唱している」を「すでに法律である」と読み違えないこと。

> 💡 ひとこと助言
> ひとこと言わせてもらえば——**技術の盾は『採集』を防ぎ、法律の盾は『収益化』を防ぐ——両者が阻むのは攻撃チェーン上のまったく異なる二つの区間だ。** 賢い蒸留者なら、演算力と書き換えで技術検知のコストをどんどん押し下げられるかもしれない。だが蒸留した重みを**対外公開され、ユーザーがおり、売上があり、法人格を持つ**製品にしようとした瞬間、彼は「マシンルームの中の透明人間」から「法廷で指名され、賠償を追及され、コンプライアンス調査を受けうる実体」へと変わる。場面3 のカナリアを復唱する生徒、グリーンリストバイアスを帯びたあの出力は、この瞬間に「統計的な嫌疑」から「法廷に提出される証拠」へと格上げされる。**これこそ成熟したメーカーが技術防護と法的抑止を組み合わせて使う理由だ——前者は君が作り上げるコストを引き上げ、後者は君がそれで稼ぐリスクを引き上げる。** エンジニアが本当に内化すべきは「どう迂回するか」ではなく、この攻防の全体図を読み解いたうえで、どの数手の先に弁護士が立っているかを明確に知ることだ——能力があれば採集できる。だが、それを使えるのは権限があってこそだ。

## 本章のまとめ

- この世代が蒸留しにくい核心は、**「リアルタイム検知＋ルート変更による汚染」の内化された防衛線**にあるとされる（真偽は要検証だが、技術ロジックは整合的）。過去の外周リスク管理より一桁手強い。
- 過去の蒸留方法論は **SeqKD**（API テキストをハードターゲットとし、`trl`／Axolotl／LLaMA-Factory でチューニング）であり、その急所はまさに「採集テキストの品質への依存」——ルート変更による汚染で一撃で崩れる。
- **検知は単一の必殺技に頼らず、四種の弱い信号の融合に頼る**：行動トラフィック指紋、出力ウォーターマーク（Kirchenbauer のグリーンリスト z 検定／Aaronson の暗号学的ウォーターマーク）、カナリア・ハニートークン事実、メンバーシップ推論（LiRA）——多層防御の本質は、壁を越える総コストを壁の向こうの物より高く吊り上げることだ。
- **戦略的非対称は「デコードの瞬間」にある**：事後検知は不可能定理（Sadasivan）に縛られるが、能動的ウォーターマークはデコードのその瞬間に証拠を植えられる——デコード権を握る者が検知権を握る。これこそ天秤が守備側に偏る根本原因だ。
- **法律の盾と技術の盾は異なる区間を阻む**：技術は「採集」コストを引き上げ、法律は「収益化」リスクを引き上げる。だが「輸出規制で即時撤去」「蒸留犯罪化」の多くは物語の継ぎ接ぎであり、現実に適用されるのは ToS／CFAA／輸出規制などの**既存**枠組みの延長だ（真偽は要検証、場面4 の ⚠️ 参照）。
- 窓の期間は極めて短く（わずか4日、4,659 件のトレースとされる）、これが次章のあのリストがなぜこれほど疎らなのかを直接決定づける。
