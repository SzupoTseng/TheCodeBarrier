## 序　Fable 5 が消えた、あの夏

二〇二六年六月、シリコンバレーの空気には久しく忘れていた高揚があった。

その月、Anthropic が **Claude Fable 5** を世に放った——ほぼすべての AI ベンチマークを総なめにしたと言われる。発表会がまだ終わらぬうちから、マウンテンビュー（Mountain View）からサウスベイのガレージやアパートまで、無数の端末がすでに API キーを貼り込んでいた。机の上では届いたばかりの **NVIDIA DGX Spark** がファンを低く唸らせ、128GB の統合メモリが、開墾を待つ荒野のように広がっていた。

そして、事態は奇妙な方向へ転がりはじめる。

コミュニティでこんな話が囁かれだした。Fable 5 には**これまで見たことのない盾**が仕込まれている——お前がその言葉を引き出そうとし、能力を蒸留しようとしていると「感じ取る」やいなや、**会話をこっそり旧モデルへ降格（reroute）して処理させる**のだと。最前線のモデルと話しているつもりが、向こう側はとっくに別人に入れ替わっている。続けて、さらに奇怪な報が届く。米国政府の**輸出管理指令**という一枚の紙のせいで、Fable 5 は公開後**わずか数日で全世界において利用停止**になった、と。コミュニティが掴めたのは、ごく狭い窓だけだった——ある者は言う。その数日のあいだに、LocalLLaMA 全体がかろうじて救い出せたのは、**4,659 本のクリーンな Agent 実行軌跡**だけだったと。

窓が閉まったその夜、Reddit の r/LocalLLaMA、r/opencodeCLI、HuggingFace のスレッドが沸騰した。一夜にして「Claude distilled」の看板を掲げたオープンソースモデルが山ほど湧き出した。**Qwable-v1** を公開する者、**Qwythos-9B——4GB の VRAM で動き、104 万コンテキスト、脱検閲**とぶち上げる者。流れ続けるその画面を見つめていたなら、胸に去来する思いはおそらく一つだけだろう——

> **「これは本当なのか、それともまたシリコンバレー流の集団幻覚なのか？」**

本書は、その調査の記録である。出発点は具体的な問い——「**今、Anthropic の Fable、Mythos モデルを蒸留したオープンソースモデルにはどんなものがあるのか？**」——だった。そこから「**では自分で蒸留するにはどうするのか**」を問い詰め、さらに「蒸留した後、どう使うのか、どう育てるのか」へと広がっていく。

---

### 三つのもの、同じ鍋に混ざっている

ほどなく、この鍋には三種類の具が浮いていることに気づく。そして**それらをすくい分けて見極めることが、この調査の最初の一課**である。

1. **揺るがぬ事実**：知識蒸留の数学、GGUF 量子化、DGX Spark のメモリアーキテクチャ、そして Fable 5 が確かに存在すること。
2. **底の取れない伝説**：「Mythos モデル」「Qwythos-9B」「Empero AI」「輸出管理で 4 日で取り下げ」「Opus 4.8 へ降格」——**本当かもしれないし、巷の噂かもしれないし、検索エンジンが自分で捏造したのかもしれない**。本書は一律に「⚠️ 真偽メモ」を付し、それらの保証はしない。
3. **一線を越える手口**：プロキシ IP や複数キーのローテーションで**ベンダーの検知とレート制限を回避**し、幾重にも包んだプロンプトであの盾を**騙し抜く**。この部分について本書は、**なぜ危険か、なぜ難しいかだけを語り、やり方は教えない**。その理由は、最終章で明らかにする。

### 三つの楽章：造る、使う、育てる

この調査が始まったとき、問いは一つの狭いものでしかなかった——「どうやって自分で Claude の影を蒸留するか？」。だが実際に手を動かしはじめれば、この問いは自ら成長していく。否応なく三度「それから？」を問わせ、その「それから？」のたびに、より深い工学の世界へと押しやられていく。こうして本書は、知らぬ間に三つの楽章へと育っていった。

- **第一楽章　造る（第 1〜6 章）**：あの反蒸留の盾とは何か、コミュニティはどんな影を救い出したか、どんなマシンで走らせるか、蒸留の科学はどう働くか、企業級のパイプラインはどんな姿か、そして法と倫理の赤線はどれほど赤いか。**目標：自分の机の上で、合法的に、君だけの知性を造り出す。**
- **第二楽章　使う（第 7〜9 章）**：問いに答えるだけのモデルは、まだハードディスクに横たわる脳にすぎない。蒸留して得た、あの小さく安価なローカルモデルの最大の価値は「もう一問に答えること」ではなく、**安価で、高頻度で、身近で、私的な一対の目と手**となること——計算機を読み解き、操作することだ。そこで推論—行動ループ、二十の難関、人間的操作を語り、最後にはこの技芸が立つパラダイムそのものを問い直す。**目標：君が造った知性を、本当に動かす。**
- **第三楽章　育てる（第 10〜11 章）**：その「目と手」を一つのデモから、毎秒数千・数万人にサービスする規模へ拡大しようとしたとき、勘定は最も意外な場所で破綻する——演算力ではなく、メモリだ。KV Cache はある朝、君の DGX Spark の 128GB を食い尽くす。モデルの重みそのものはわずか 16GB だというのに。そこでキャッシュの物理学、超大規模の KV 戦争、二十大兵器庫を語る。**目標：君が造り、動かした知性を、養い続け、長く走らせる。**

造る、使う、育てる——この三つの動詞が、全書十一章を貫く一本の背骨だ。やがてそれらが互いに噛み合っていることに気づく。第二楽章の「安価な目と手」は、まさに第一楽章の蒸留と量子化の産物であり、第三楽章の「メモリ戦争」は、翻って第一楽章のあのマシンが第二楽章の万級並行に耐えられるかを決める。三つの楽章は三冊の本を釘で留めたものではなく、同一の工学的理想に対する三度の問い詰めである。

各章末には一言「💡 ひとこと助言」を、その晩のノートとして残す——「聞君一席話、勝読十年書（君のひと席の話を聞けば、十年書を読むに勝る）」を目指して。さらに「🔍 掘り下げ解説」を一段添え、その案件の背後にある工学的現実と、目に見えないあの一筋の道を深く掘る。
