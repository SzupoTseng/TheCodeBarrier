# 付録F　エージェント操作（Computer Use）ツールと論文クイックリファレンス

> 第7〜9章に登場した「Computer Use / GUIエージェント」エコシステムを、主題ごとに一枚の対照表へまとめる。**本付録は二種類を区別する**。標準的な視覚グラウンディングモデル、オープンソースのエージェントフレームワーク、評価ベンチマークとサンドボックス基盤の多くは**実在し、公開された論文またはプロジェクト**である（各自で検証可能）。一方、第7〜9章の物語に登場するいくつかの「Google I/O 2026 製品名」は**未検証**であり、文末の ⚠️ ブロックに明示的に標記した。既成事実として引用しないこと。

## F.1　視覚グラウンディング / 画面理解モデル

| 名称 | 一言でいう位置づけ | 出典 / ソース |
|---|---|---|
| **Set-of-Mark（SoM）** | スクリーンショット前にUI要素を分割し数字ラベルを重ねることで、「座標予測」を「選択問題」に変える | Microsoft Research、2023（GPT-4V 視覚グラウンディング）✅ |
| **SAM 2** | 画像/動画の汎用セグメンテーションモデル。SoMのUI要素検出の前処理としてよく使われる | Meta、2024 ✅ |
| **CogAgent** | GUI操作専用にファインチューニングした18BのVLM。高/低解像度の二系統チャネル | THUDM、CVPR 2024 ✅ |
| **ScreenAI** | UIレイアウト図と底層の意味木（Accessibility/DOM）を同時に読み解くVLM | Google、2024 ✅ |
| **Ferret-UI** | モバイル端末（iOS/Android）のUI理解小型モデル。要素分類/OCR/グラウンディング | Apple、2024 ✅ |
| **NaViT** | 任意のアスペクト比/解像度に対応するViTのパッキング学習法（マルチスケール視覚ピラミッド概念の源流） | Google、2023（概念）✅ |
| **ViT（Vision Transformer）** | 画像をpatchに切ってtokenとして扱う視覚バックボーン。上記モデルの共通基盤 | Dosovitskiy et al., 2020 ✅ |

## F.2　Web / GUIエージェントフレームワーク（オープンソース）

| 名称 | 役割 | 出典 / ソース |
|---|---|---|
| **BrowserUse** | Playwright + LLM で駆動し、ページの操作可能要素を抽出してから意思決定するブラウザエージェント。リトライ/バックトラックの堅牢性が高い | GitHub `browser-use`（オープンソース）✅ |
| **Skyvern** | VLMで駆動するブラウザ自動化。XPath/セレクタのハードコード不要 | GitHub `Skyvern-AI`（オープンソース）✅ |
| **WebVoyager** | エンドツーエンドのマルチモーダルWebエージェント（スクリーンショット+SoM）。同時に評価セットでもある | 論文、2024 ✅ |
| **Mind2Web** | 汎用Webエージェントのデータセット/ベンチマーク（F.3 も参照）| OSU NLP、2023 ✅ |
| **Code-as-Action / CodeAct** | LLMに実行可能なコード（JS/Python）を直接アクションとして出力させ、視覚的なゆがみを免れる | 論文 *Executable Code Actions…*、UIUC 2024 ✅ |

## F.3　評価ベンチマーク（Benchmarks）

| ベンチマーク | 何を測るか | 規模 / ソース |
|---|---|---|
| **OSWorld** | 実OSレベルのエンドツーエンド操作（Office/ターミナル/GIMP など） | 369タスク、2024、業界公認で最難 ✅ |
| **WebArena** | セルフホストのWebサイト（EC/フォーラム/Git/CMS）における機能的なWebタスク | 812タスク、CMU 2023 ✅ |
| **VisualWebArena** | WebArenaの視覚強化版。画像内容の理解が必要 | 2024 ✅ |
| **Mind2Web** | 137の実在Webサイトにおける汎用Web操作 | 2,000+タスク、2023 ✅ |
| **AndroidWorld** | Android実機環境の動的タスク。パラメータ化された報酬を含む | 116タスク、Google 2024 ✅ |

## F.4　インフラ / サンドボックス

| 名称 | 役割 | ソース |
|---|---|---|
| **Browserbase** | マネージド型のヘッドレスブラウザクラスタ。Session/Proxy/アンチクローリング対策を処理（オープンソースの Stagehand フレームワークも推進）| 商用（YCスタートアップ）✅ |
| **gVisor** | アプリケーション層のユーザー空間カーネルサンドボックス。syscallを傍受し信頼できないコードを隔離 | Google、オープンソース ✅ |
| **Firecracker MicroVM** | <125ms 起動の軽量KVM。エージェント1つにつき隔離環境1つ | AWS、オープンソース ✅ |
| **Playwright** | クロスブラウザ自動化（DOM操作 / JS注入 / スクリーンショット） | Microsoft、オープンソース ✅ |

## F.5　プロトコル / セキュリティ / アーキテクチャ概念

| 用語 | 一言での説明 | ソース |
|---|---|---|
| **MCP（Model Context Protocol）** | モデルが標準プロトコルで外部ツール/データを呼び出すためのオープンインターフェース | Anthropic、2024 ✅ |
| **Constitutional AI** | 一連の「憲法」原則で自己批判/アライメントを行う。転じてOS動作傍受のガードレールへ応用 | Anthropic ✅ |
| **State Space Models / Mamba** | 線形計算量の系列モデル。長系列記憶の代替アーキテクチャ | Gu & Dao、2023 ✅ |
| **ReAct** | 推論（Reason）と行動（Act）を交互に行うエージェントのパラダイム | Yao et al., 2022 ✅ |
| **Chain-of-Thought（CoT）** | モデルに一歩ずつ推論させてから答えさせる（プランニング層の中核） | Wei et al., 2022 ✅ |
| **[0,1000] 座標正規化** | クリック座標を 0〜1000 の整数区間に正規化し、解像度に依存しないようにする | CogAgent/Qwen-VL などの汎用手法 ✅ |
| **Location Tokens（位置 token）** | 特殊tokenで座標/バウンディングボックスを直接表現する、視覚グラウンディングの定番手法 | Pix2Seq/CogAgent/Ferret ✅ |
| **Accessibility Tree 二重流アライメント** | 視覚スクリーンショット + アクセシビリティ木のクロスアテンション。幻覚時は意味木で座標を補正 | ScreenAI のアプローチ ✅ |
| **HITL / MFA Break Protocol** | CAPTCHA/MFAに遭遇したらVMを凍結し、人手に引き継いでからシームレスに解凍 | エンジニアリング実践（概念）✅ |
