# 第四部　蒸留の原理

# 第4章　蒸留の科学：「知性」を煮詰めて一瓶のエッセンスにする

> シリコンバレーの深夜のフォーラムに、ある人物が訓練曲線の画像を貼ってこう問うた。「なぜ私の生徒モデルはベンチマークのスコアは上がったのに、実際に使ってみると答えを丸暗記しただけの劣等生のようなのか？」その下で最も「いいね」を集めた返信は、たった一文だった。「**お前が蒸留したのは答えであって、分布じゃないからだ。**」——この一言こそ、本章で徹底的に解きほぐすものである。

> 本章は純粋な機械学習の教育コンテンツである。知識蒸留（Knowledge Distillation）は公開され、成熟した、教科書級の技術であり、**使用権を持つ**オープンウェイトモデルを蒸留することは完全に正当である。本章ではこの分野でよく使われる用語（ソフトラベル、KL、軌跡反転、CoTアライメント、量子化）を、**その背後の数学とオープンソースツール**ごと一気に語り尽くす。

## 場面1　ハードラベル vs ソフトラベル：なぜ「確率分布」は「答え」より価値があるのか

**背景**：知識蒸留の最も核心的な観念は、「教師が生徒に一体何を伝えるのか」という点に潜んでいる。

**問題**：教師が「猫の英語は？」に答えて最終的に「cat」を出力する。生徒が学ぶべきは「答えはcat」なのか、それとも別の何かなのか？

**二つの学び方の数学的形式**：

- **ハードラベル（Hard Target）**：one-hotの最終的な答えだけを学ぶ。損失は標準的な交差エントロピー `CE(student, "cat")`。生徒は「正しい」ことだけを知り、「なぜ正しいのか、他の選択肢はどれだけ間違っているのか」を知らない。
- **ソフトラベル（Soft Target / Logits）**：教師が出力する**完全な確率分布**を学ぶ——`cat 92%, kitty 5%, feline 2%, dog 0.01%...`。

**Hinton蒸留損失の標準形式**（これが本章全体の数学的骨格である）：

```
L = α · CE(y_true, softmax(z_s))              ← ハードラベル項
  + (1-α) · T² · KL( softmax(z_t / T) ‖ softmax(z_s / T) )   ← ソフトラベル項
```

- `z_s`、`z_t`：生徒と教師のlogits（softmaxを通す前の生のスコア）；
- **温度T（軟化）**：温度付きのsoftmaxはlogitsを平らに均す——

```
p_i(T) = exp(z_i / T) / Σ_j exp(z_j / T)
```

  T=1は元のsoftmax；T→∞で分布は一様に近づく（最大限に長尾を露出させる）；T→0でone-hot（argmax）に退化する。Tを上げることは、`dog 0.01%`のような指数関数によって0近くまで圧縮された「暗黙知（ダークナレッジ）」を、学習可能な量級まで再び増幅することに等しい——**蒸留が本当に伝えたいのは、あの92%というピークではなく、ピークの外側にあるほとんど見えないほどの相対的な構造なのだ。**
- **なぜ `T²` を掛けないといけないのか（最も漏らされやすく、漏らすと調整が効かなくなる）**：ソフトラベル項の生徒logitに対する勾配は、その量級が `1/T²` に比例する：

```
∂L_soft/∂z_s,i = (1/T) · ( softmax(z_s/T)_i − softmax(z_t/T)_i )
高いTのもとで指数に一次展開を施すと、全体の量級 ∝ 1/T²
```

  したがって、ひとたびTを1から4に上げると、ソフトラベルの勾配は約16倍に縮み、ハードラベル項に完全に飲み込まれてしまう。`T²` を掛け戻すのは、「ソフト・ハード両項の相対的な重みがTによって漂わない」ようにするためである——これはHinton, Vinyals & Dean（2015,『Distilling the Knowledge in a Neural Network』）の原文が名指しで指摘しているにもかかわらず、実装者に最も漏らされやすいディテールである。
- **慣例の小注**：`α` と `1−α` がどちらの項に掛かるかは単なる命名の慣例にすぎない——少なからぬ文献は `α` をKLソフトラベル項に、`1−α` をCEに掛けており、上式と数学的に等価である。本当に間違えてはならない二つのことは、**`T²` は必ずKL項に掛ける**こと、そして**両項の量級をそろえる**ことである。

**なぜソフトラベルが魂なのか**：あの分布の中には教師の**暗黙知（Dark Knowledge）**が潜んでいる——「kittyはdogより答えに近い」というこの**相対的で曖昧な判断**こそ、大規模モデルが数兆トークンを費やしてようやく育てた世界観なのだ。

> 🧠 蒸留の中核的な直感
> **ハードラベルは「結論」を教え、ソフトラベルは「センス」を教える。** 結論だけを丸暗記した生徒は、見たことのない問題に出会うとお手上げになる。教師のセンス（どの選択肢がより起こりやすいか、どれだけ的外れに間違っているか）を学んだ生徒だけが、一を聞いて十を知る能力を持つ。

**商業蒸留を強制的に格下げさせる残酷な現実**：ソフトラベルは教師がlogitsを吐き出すことを必要とするが、**クローズドソースの商業APIは通常、完全なlogitsを出さない**——手に入るのはテキストだけ（せいぜいtop-5のlogprobs）。だからクローズドソースモデルの蒸留は、多くの場合**系列レベル蒸留（SeqKD）**へと後退する：教師の「最も起こりやすい出力系列」をハードターゲットにしてSFTを行う。**これこそ第2章「形は似ても神は似ず」の数学的な根源である——分布が手に入らなければ、答えを蒸留するしかない。**

> 💡 ひとこと助言
> 君と語り合って一席。**最良の教師はただ答えを与えるだけでなく、「他の答えがどこで、どれだけ間違っているか」をも見せてくれる——惜しむらくは、最良の教師ほど下書きの紙を鍵のかかった引き出しにしまってしまう。** ソフトラベルが蒸留の魂たる所以は、「不確実性」そのものが知識だからだ：あるモデルが `cat` に92%を与えて100%を与えず、`kitty` に5%を残す、この「余地を残す」態度こそ、それが世界を見てきた証である。クローズドソースAPIがlogitsを鍵で封じるとき、封じているのは出力ではなく「教師の躊躇」なのだ——そして真の知性は、まさにその躊躇の重みの中にこそ潜んでいる。これこそが次世代の防御の核心が「分布を隠す」ことにある理由だ：**お前は人の答えを複製できても、その人が間違えるときの「さじ加減」は複製できない。**

> 🔍 掘り下げ解説──暗黙知はなぜ「数兆トークンを費やしてようやく育つ」ものなのか
> ソフトラベルの価値を徹底的に語るには、一つの定量的な問いに立ち返らねばならない。**教師のあの分布の中には、one-hotラベルが持たない「情報」が一体どれだけあるのか？** 情報理論が与える答えは明快だ——one-hotラベルのエントロピーは0（百パーセント確実）だが、教師が「猫の英語」に対して `cat 92% / kitty 5% / feline 2% …` という分布を出すとき、そのエントロピーは正である。この余剰のエントロピーこそ、Hintonが2015年に言った暗黙知だ：クラスとクラスの間の**相対的な類似度構造**（kittyはdogより近く、felineはkittyよりさらに書き言葉的）である。標準的な教師あり学習は、各サンプルにつき上限 `log(クラス数)` ビットの信号しか与えないが、ソフトラベルは一度に分布全体を与える。これは各トークンの監督が一桁分増幅されることに等しい——これこそ「同じデータ量でも、蒸留はゼロからの教師あり学習より速く、サンプル効率がよい」ことの数学的な根因である（Hintonはこれを「教師はただ答えを与えるだけでなく、それぞれの間違った答えが各何点減点される理由まで与えてくれる」とたとえた）。だがここで一つのよくある誤解を打ち砕いておく：**暗黙知は多ければ多いほどよいわけではない。** Tを開きすぎると、教師自身ですら確信を持てない長尾のノイズ（あの0.001%の尾）まで一緒に増幅して生徒に食わせてしまい、かえって生徒に教師の「幻覚の残渣」をフィットさせることになる。実務上Tは2〜4の範囲に置き、`T²` 補償を併せるのが、コミュニティが繰り返し検証してきたスイートスポットだ——**蒸留すべきは教師の判断力であって、その過剰な躊躇ではない。**

## 場面2　Forward KL vs Reverse KL：生徒は「全面模倣」すべきか「善きを択びて固く守る」べきか

**背景**：上のKL項は、その方向にこそ実は大きな学問がある。よく言及される「on-policy蒸留」「軌跡反転」は、背後ではいずれもKLの方向と関係している。

**問題**：`KL(teacher ‖ student)` と `KL(student ‖ teacher)` は、どこが違うのか？

- **Forward KL（`KL(teacher ‖ student)`、標準蒸留）**：生徒に「教師のすべてのモードをカバーせよ」と迫る——教師に何らかの可能性があれば、生徒もすべて面倒を見なければならない。結果は **mode-covering（モード被覆）**：生徒は「八方美人だが凡庸」になり、支えきれないところでも無理に学ぼうとして、幻覚じみた平均化を生み出す。
- **Reverse KL（`KL(student ‖ teacher)`、MiniLLMの選択）**：生徒に「自分が自信のあるところでのみ教師に近づけ」と迫る——**mode-seeking（モード探索）**：生徒は教師の最も主要ないくつかのモードに火力を集中して学び、出力はより鋭く、デタラメが少なくなる。特に**容量が教師よりはるかに小さい**蒸留（小モデルが大モデルを学ぶ）に向いている。

**二つの方向の数学は、「誰を加重の基準分布とするか」が違う**：

```
Forward KL:  KL(P_teacher ‖ Q_student) = Σ_x P(x) · log( P(x) / Q(x) )
Reverse KL:  KL(Q_student ‖ P_teacher) = Σ_x Q(x) · log( Q(x) / P(x) )
```

- **Forward KLは `P`（教師）で加重して総和を取る**：教師がある箇所で確率を持つ（`P(x)>0`）とき、生徒がそれを0に圧縮すると（`Q(x)→0`）、`log(P/Q)→∞`——**「教師のいかなるモードも漏らすこと」を重く罰する**。それゆえ生徒はzero-avoiding（どこにも空白を残さない）になり、結果はmass-coveringの平均化、つまり複数のモードを一塊に塗りつぶしてしまう。
- **Reverse KLは `Q`（生徒）で加重する**：生徒自身が確率を持つところでのみ損失に入り、教師の長尾（生徒の `Q(x)≈0` の箇所）は損失にほとんど寄与しない——**生徒は教師の希少なモードを大胆に捨てることができ**、質量を主峰に集中させる、すなわちzero-forcing / mode-seekingである。

**これこそがon-policy蒸留の価値だ**：MiniLLM、GKD（Generalized KD）は、**生徒にまず生成させ、教師が生徒の出力を採点／修正する**もので、本質はreverse KLを使って「生徒自身の分布」の上でアライメントすることである——静的なテキストを死に物狂いでかじる（forward KL / SeqKD）よりもノイズに強く、過学習しにくい。代償は、教師をより高頻度で呼び出す必要があり、レート制限と検知（第1章）に正面から衝突することだ。

> 🔍 掘り下げ解説──KL方向の選択は、「忠実さ」と「鋭さ」の間のエンジニアリング的トレードオフ
> この一見抽象的な数学的ディテールが、蒸留して得られるモデルの「人格」を直接決める。**Forward KLは「お人好し」を育てる**——教師のあらゆる可能な回答を模倣しようとし、自分の能力が及ばないところで「分かったふり」を強いられ、出力は平均的で安全だが、たまにデタラメを言う（自分でも支えきれないモードまでカバーしているから）。**Reverse KLは「専門家」を育てる**——教師の長尾を覆うことを諦め、最も核心的なモードに集中して学び、出力はより断固として幻覚が少ないが、教師の一部の稀有な才能を取りこぼす。「9BがClaudeを学ぶ」ような**容量が懸絶した**蒸留では、reverse KLがほぼ常に優れている——なぜなら小モデルはそもそも大モデルの全モードを収めきれず、被覆を強いる（forward KL）と、支えきれないところで嘘をつくよう迫るだけだからだ。これは一つの反直感的な現象も説明する：**ときに「学んだことが少ない」生徒のほうが「使い勝手がよい」**のは、それが誠実に、できるところでだけ力を発揮するからだ。このforward/reverseの選択には堅実な論文の裏付けがある：**MiniLLM（Gu et al. 2023,『Knowledge Distillation of Large Language Models』）** は明確にreverse KL + 方策勾配を用いて大モデルを蒸留し、それが「小さな生徒」に対して標準的なforward KLより優れることを論証した；**GKD（Agarwal et al. 2024,『On-Policy Distillation of Language Models』）** は方向（forward/reverse/一般化JSD）と「誰の軌跡の上で学ぶか」を一緒にパラメータ化した（次節参照）。ツールの面では、MiniLLMには公式実装があり、TRLの `GKDTrainer`、`torchtune` もオンライン蒸留に対応済みだ——だがそれらがいずれも教師への頻繁なアクセスを必要とすることを忘れてはならない。これはクローズドソースAPIにとってコストとコンプライアンスの二重の高い壁である。

## 場面3　系列レベル蒸留とon-policy：「トークン単位のアライメント」が自己回帰の呪いに衝突するとき

**背景**：場面2のKLは「トークン単位で、教師のデータの上で」計算されたものだ——これは自己回帰言語モデルに一つの古典的な病巣を潜ませている。それこそKim & Rush、GKDという二つの手がかりが本当に対処している問題だ。

**トークン単位蒸留の呪い——露出バイアス（Exposure Bias）**：標準的なtoken-level蒸留は **teacher forcing** を用いる：各ステップで教師が正しく書いたプレフィックスを食わせ、生徒は「次のトークンの分布」を予測するだけだ。だが推論時、生徒が食べるのは**自分が生成したプレフィックス**だ——あるステップでひとたびずれると、その後は訓練時に一度も見たことのないプレフィックスの上に立つことになり、誤差が雪だるま式に膨らむ。**訓練分布（教師の軌跡）とテスト分布（生徒自身の軌跡）が合わない、これが露出バイアスである。** 各トークン上でKLを極限まで圧縮しても、文全体の軌跡は依然として外れていく——なぜなら損失は「生徒がたどり着く状態」の上で一度も計算されたことがないからだ。

**Kim & Rushの系列レベル蒸留（Sequence-Level KD, 2016）**：トークン単位で確率をアライメントするより、教師に**ビームサーチで最も起こりやすい出力系列全体を生成させ**、それらの系列をハードターゲットにして生徒を訓練するほうがよい。直感的には、それは「系列空間」において教師のモードを近似する——トークンレベルのソフトアライメントを、系列レベルの「最も起こりやすい文全体を学ぶ」に置き換えたのだ。**これこそクローズドソースAPI蒸留が次善の策とする理論的根拠である：logitsが手に入らなくても、せめて教師の最も起こりやすい出力系列をハードターゲットにできる。** 代償は二つ：系列レベルはトークンレベルの暗黙知（場面1）を捨てており、かつそれは依然としてoff-policyである——学ぶのは教師の軌跡であって、生徒自身のものではない。

**On-policy / GKD——「誰の軌跡の上で学ぶか」をひっくり返す**：MiniLLM（Gu et al. 2023）と **GKD（Generalized KD, Agarwal et al. 2024）** の核心的な動作は——**生徒にまず自分で生成させ、その後教師が生徒の出力を採点／KLを計算する**ことだ。こうすれば訓練分布は自然とテスト分布に等しくなり（どちらも生徒自身のもの）、露出バイアスが根こそぎ取り除かれる。GKDはさらにそれを一つの調整可能な一般化ダイバージェンスとして書き下す：

```
L_GKD = E_{y ~ 混合(生徒π_s, 教師データ)} [ D( p_teacher ‖ p_student )(y) ]
  混合比 λ：λ→0 純 on-policy(生徒の軌跡) … λ→1 純 off-policy(教師データ)
  ダイバージェンス D：forward-KL / reverse-KL / 一般化 JSD(β) を選べ、両者の間を滑らかに補間
```

GKDの貢献は何か固定のレシピではなく、**「誰の軌跡の上で学ぶか（on/off-policy）」と「どの方向のダイバージェンスを使うか（forward/reverse）」を二つの直交したつまみに分解した**ことにある——容量が懸絶していればreverse-KL + 高いon-policy比率へ寄せ、容量が近ければforward-KL + 教師データを多用する。代償は、on-policyが訓練ループの中で**教師に繰り返し生徒のリアルタイム出力を採点させる**必要があることで、クローズドソースAPIにとってはコストとレート制限の二重の高い壁（第1章）だ。これがオープンソースの影武者の多くが安価なoff-policy SeqKDに留まる理由でもある。

> 🔍 掘り下げ解説──「誰の軌跡の上で学ぶか」は「どれだけ似ているか」よりも上限を決める
> 初心者が蒸留を調整するとき、目はすべて損失関数（KLを使うかCEか、温度をいくつにするか）に釘付けになる；熟練者はまずもっと上流の問いを立てる：**この訓練系列の束は、教師が書いたのか、それとも生徒自身が書いたのか？** このon-policy / off-policyの区別は、損失関数の選択よりも上限を決定づける。理由は極めて物理的だ：モデルはデプロイ後**自分の分布**の中で生きているのに、もし**教師の分布**の上でしか訓練されていなければ、それは「一生コーチが球を出してくれる場所で練習してきたのに、試合では自分でサーブしなければならない」のと同じだ——球筋がひとたびコーチの出してくれた範囲を外れれば、どう救えばいいか学んだことがない。on-policy（GKD / MiniLLM）の本質は、練習場を試合場に移すことだ：**生徒が自分で犯すであろう間違いの上で、教師にその場で正してもらう**、そうして学ぶのが「私がたどり着くであろう状態のもとでどうすべきか」だ。これは業界で繰り返し現れる一つの現象も説明する——**同じ生徒、同じ教師でも、訓練データを「教師生成」から「生徒生成 + 教師採点」に換えるだけで、下流の性能がしばしば一段階跳ね上がる**、しかも損失関数は一字も変えていない。代償は正直にそこにある：on-policyは訓練中に高頻度で教師にアクセスする必要があり、高く、遅く、かつクローズドソースの教師に対しては直接ToSとレート制限に衝突する。**安価なoff-policyと高価なon-policyの間に横たわるのは賢さの差ではなく、お前が（合法的に、持続的に）教師を生徒に付き添わせ、ともに間違え、ともに直させられるかどうかの差なのだ。**

## 場面4　CoTアライメントと軌跡反転：生徒に「丸暗記」ではなく「しっかり考えさせる」

**背景**：この世代の蒸留の重点は**思考連鎖（CoT）**だ——モデルが `<thinking>` ブロックの中で行う推論過程；そして繰り返し現れる一つの謎めいた言葉：**軌跡反転（Trace Inversion）**である。

**CoTアライメント損失（CoT Alignment Loss）のやり方**：思考連鎖ブロックの中のトークンに**重み付け**する（例えば重み2.0）。これは訓練時に生徒にこう告げるに等しい：「**お前が `<thinking>` の中に書く一字一字は、他の場所の二倍の重みがある——本気で考えろ。**」これは生徒に、答えを出す前に**十分な計算量を思考に配分する**こと（compute-over-thinking）を迫る。問題を見るなり条件反射で答えを噴き出すのではなく。

**軌跡反転の本質**：教師の推論ステップを**逆順にする、あるいは再構造化して**から生徒に食わせることだ。教師の軌跡が「原因A→B→C→答えD」だとすると、反転後は「答えD、逆にたどって依存するC、CはBに依存し……」となる——**生徒に各中間ノードの因果を理解させ、「問題→答え」の近道を丸暗記させない。**

**理由蒸留（Rationale Distillation）——CoTを「もう一段のテキストを余分に食わせる」ことではなく「マルチタスク監督」にする**：Hsieh et al.（2023,『Distilling Step-by-Step!』）はCoT蒸留により鋭い形式を与えた——教師の推論連鎖をより長い丸暗記用テキストとして生徒に詰め込むのではなく、それを**第二の監督タスク**として扱う：訓練時、小モデルに「答えを予測する」ことに加えて、**その答えの理由（rationale）を同時に出力させる**ことを要求し、二つの損失を並行して加重する。論文の鍵となる発見は——**理由を監督として使うことで、小モデルはより少ないデータで、さらには小さな体格でも、最終的な答えだけを学ぶ標準的なファインチューニングを上回れる**ということだ。これはCoTを「プロンプトのテクニック」から「訓練シグナル」へと格上げした：理由は人に見せる飾りではなく、生徒に「答えがどう推論されたか」を重みの中に内面化させる監督のテコなのだ。これは場面1とも呼応する——**理由とは系列版の暗黙知であり、教師の「思考の分布」をテキストへと平らに展開することで、logitsが手に入らない生徒にもその一部を間接的に盗ませる。**

**併用するデータ拡張——意味的言い換え（Synthetic Paraphrasing）**：高温サンプリング（例えばTemperature=0.85）で同じ一本の良質なデータを3通りの言い方に書き換え、汎化を広げ、過学習を防ぐ。ツールの面では **distilabel（Argilla）** がこの種の合成／言い換えデータを作る主流のオープンソースフレームワークだ。

> ⚠️ 真偽メモ──「軌跡反転」には二つの読み方があり、一つは無害、一つは一線を越える
> 本節で語る「軌跡反転」は**データ拡張の意味での**ものだ——教師が**自分で出力した推論ステップ**を逆順／再構成し、生徒に因果を学ばせて丸暗記させない、これは完全に正当である。だが同じ言葉には**第二の、より攻撃的な読み方**が潜んでいる：教師の**最終出力から、それが見せない入力を逆推する**——隠されたシステムプロンプト、プライベートなfew-shotテンプレート、さらには教師の内部指令を再構築する（output → prompt 反転）。この第二のものはもはや「推論構造を並べ替える」ことではなく、**クローズドソースの教師に対するリバース抽出**であり、第1章・第6章が警告する「ベンダーの隠された資産を吸い出す」のと同じ一線である。本書のこれに対する立場はM1と一致する：**その存在とリスクを記述するのみで、いかなる反転手法やプロンプト再構築スクリプトも提供しない**。「私の手元にあるこの推論連鎖を並べ替える」ことと「他人が鍵をかけたプロンプトを逆推する」こと、この二つを必ず区別してほしい——前者は訓練テクニックであり、後者は攻撃行為である。

> 💡 ひとこと助言
> 君と語り合って一席。**蒸留の敵はそもそも「データが少なすぎる」ことではなく、「データが似すぎている」ことだ。** 蒸留データは生まれつき同質である——同じ一人の教師、同じ一つの文体、同じ一束の問題タイプ——モデルはひとたび「順序を暗記する」ほうが「論理を理解する」より省力だと気づくと、近道を取って過学習する。軌跡反転、意味的言い換え、CoT重み付け、この三つの手は表面上は各々異なるが、骨の髄では同じ一つのことだ：**データの表面的な規則性を人為的に破壊し、「丸暗記の近道」を塞ぎ、モデルを退路なきところまで追い詰めて、底層の構造を学ばせるしかない状態にする。** これは人に教えるときも成り立つ——例題のステップをかき混ぜて生徒に並べ替えさせ、正しく並べ替えられた者こそが本当に理解している。**よい訓練とは、より多くの似た問題を食わせることではなく、一問ごとに「前の一問のパターンでは解けない」ようにすることだ。**

> 🔍 掘り下げ解説──軌跡反転の機械学習の系譜における位置づけと、よく見落とされる一つの落とし穴
> 「軌跡反転」を機械学習の大きな系譜に戻すと、それは **データ拡張（Data Augmentation）+ 正則化（Regularization）** の一つの知的なバリアントであり、コンピュータビジョンで「反転、切り抜き、ノイズ付加」で画像の過学習を防ぐのと同源の思想だ——ただ「ピクセルを摂動する」のを「推論構造を摂動する」に換えただけだ。それは意味的言い換え、棄却サンプリングと同じ一つのファミリーに属し、目的はいずれもデータの表面的な規則性を破ることだ。だがここには**よく見落とされる一つの落とし穴**がある：摂動は**論理的有効性を保たねばならない**。画像は反転しても依然として猫だが、推論ステップは逆順にした後、もし因果依存を破壊すれば（Bは実はCの前にあって初めて成立する）、生徒に食わせるのは**論理的に誤った「偽CoT」**だ——生徒は誤った推論構造まで一緒に学び込み、かえって中毒する。だから高品質な軌跡反転には一つの検証関門が必要だ（別のモデルやルールで反転後の論理が依然として整合しているか検査する）。これこそ第5章のパイプラインがそれを「二重アライメントとフィルタリングモジュール」に組み込み、軽々しくやらない理由でもある。**データ拡張の一線は「拡張は意味的真理値を変えてはならない」だ——摂動の形式は許されるが、事実の歪曲は許されない。**

## 場面5　選好蒸留とDPO：「テキスト」しか手に入らなくても、「センス」を盗めるか

**背景**：場面1で語った残酷な現実——クローズドソースAPIはlogitsを出さず、ソフトラベル蒸留はSeqKDのハードターゲットへと退化する。だが「logitsがない」は「答えしか学べない」と同義ではない。センスを盗むもう一本の道がある：**選好（Preference）**だ。

**核心的な発想の転換**：教師の「センス」はlogitsの中だけでなく、その**選択**の中にも潜んでいる——同じ問題に教師が複数の回答を生成する、あるいは教師に二つの回答から一つを選ばせる、この「AはBより良い」という相対的な判断そのものが暗黙知のもう一つの投影であり、しかもそれは**テキストインターフェースだけで取得できる**。

**DPO（Direct Preference Optimization, Rafailov et al. 2023）**：従来のRLHFはまず報酬モデルを訓練し、その後PPOで強化学習する必要があり、重くて不安定だ。DPOは証明した：選好ペア `(x, y_w 勝, y_l 敗)` が与えられれば、**報酬モデルを飛ばして、直接方策に対して分類式の損失を一つ作れる**：

```
L_DPO = − E_(x, y_w, y_l) [ log σ( β · ( log[π_θ(y_w|x)/π_ref(y_w|x)] − log[π_θ(y_l|x)/π_ref(y_l|x)] ) ) ]
  π_θ   = 生徒(訓練対象)
  π_ref = 凍結した参照(通常 = SFT 後の生徒)
  β     = KL 制約の強度, σ = sigmoid
```

直感：**「勝った回答」の参照モデルに対する対数確率を引き上げ、「負けた回答」のそれを引き下げる**。そして `β` は手綱のように、生徒が選好におもねるあまり参照モデルから離れすぎないようにする（式の中に逆向きのKL制約が暗に含まれている）。

**選好蒸留（Preference Distillation）の繋ぎ方**：教師を「審判」または「生成器」として使い、選好ペアを作る——教師に生徒（または教師自身）の複数の出力を順位付けさせ、chosen / rejectedを選び出させ、それをDPOに食わせる。こうして生徒が学ぶのは「教師がどんな字を書いたか」ではなく、「**教師は二つともよどみない答えの間で、何を根拠にこの一方を偏愛したのか**」だ——文体、語気、安全境界、判断の尺度、これらSeqKDのハードターゲットでは伝わらないものを、選好ペアは伝えられる。これは場面2のreverse-KL「mode-seeking」の精神のもう一つの実装でもある：DPOの `β`-KL制約は生まれつきreverse方向で、生徒を教師が選好するモードへと**鋭化**させ、全部を覆わせない。

> 💡 ひとこと助言
> 君と語り合って一席。**最も盗みにくいのはそもそも「教師が何を言ったか」ではなく、「教師が二つとも正しい答えの中で、なぜこの一方を選んだか」だ——答えは写せても、取捨は写せない。** 選好蒸留の深さは、「logitsが手に入らない」というこの壁を迂回し、別の方向から同じ一つの暗黙知をすくい上げることにある：お前が私に一万回「AはBより良い」を見せれば、私はお前のあの善し悪しを測る物差しを逆推できる。これは弟子を取るときも成り立つ——一人の弟子のセンスを本当に形作るのは、師匠が正しく作り上げた完成品ではなく、師匠が弟子の目の前で、あの「よさそうにも見えるが、しかしどうも足りない」版を**却下する**ことだ。**作れる人は多いが、何を作るべきでないかを知り、しかもなぜかを言える人は少ない——そして後者こそ、センスのすべてである。** だが選好が判断と安全の尺度を伝えられるからこそ、それは同時に諸刃の剣でもある：教師の選好ペアで安全境界を**消し去る**（「拒否＝負、従う＝勝」の選好ペアを大量に作ってDPOする）のは、第2章が警告した脱検閲の選好層における焼き直しだ——「センス」を伝えられる経路は、逆に「無センス」も伝えられる。

## 場面6　語彙横断蒸留と量子化：二つのエンジニアリングの硬い骨

**背景**：よく素通りされるが、実戦では必ず衝突する二つの硬い問題：教師と生徒の**語彙が一致しない**ときどう蒸留するか？蒸留した良いモデルをどうグラボに押し込むか？

**硬い骨その一：トークナイザの不整合（Vocabulary Mismatch）**

教師（Claude）と生徒（Qwen/Llama）のトークナイザはほぼ同じになりえない——同じ一文が切り分けられるトークン数も境界も異なり、logitsは座標すら合わない。解法：

- **語彙マッピング + 動的アライメント**：トークンマッピング表を作り、教師の確率質量を生徒の語彙へ再配分する；
- **ULD（Universal Logit Distillation）/ 語彙横断KL**：最適輸送（optimal transport）または周辺アライメントを用いて、異なる語彙の間でKLダイバージェンスの補償を行う——これは近年のモデル横断蒸留の活発な研究方向だ。
- **退路**：語彙がどうしても合わなければ、logit蒸留を諦め、純テキストのSeqKDに戻る（また「形は似ても神は似ず」に戻る）。

**硬い骨その二：量子化（Quantization）——蒸留した良いモデルをあの128GBに押し込む**

- **手法のスペクトル**：訓練後量子化（PTQ）の主流には **GGUF K-quants（llama.cpp）、GPTQ（AutoGPTQ）、AWQ（AutoAWQ）** があり；訓練中／QATとQLoRAファインチューニングには **bitsandbytes（NF4）** を使う。
- **混合量子化（玄人の手筋）**：**モデル全体に同じ量子化レベルを使わない**——精度に敏感な**アテンション層（Attention）はQ8_0で守り**、体積が大きく相対的に圧縮に耐える **MLP/FFN層はQ4_K_Mに圧縮する**。なぜならアテンションは「思考の関節」であり、壊すとモデルが馬鹿になる；MLPは「知識の倉庫」であり、少し圧縮しても影響が小さいからだ。
- **imatrix校正**：量子化前に少量の校正コーパスで**重要度行列**を計算し、低ビットが優先的に肝心の重みを守れるようにする——同じQ4の体積でも、品質が明らかに高くなる。
- **KV Cache量子化（FP8/INT8）**：長文脈の場面で文脈のメモリも圧縮する（vLLM、llama.cppがすでに対応）。これは第3章の長文ボトルネックの真の解だ。

**量子化の科学：PTQ vs QAT、そして名前を言えるべき四つの手法**

量子化の本質は、高精度の重み `w`（FP16）を低ビット整数グリッドへマッピングすることだ：`w ≈ scale · round(w/scale) + zero_point`——`scale`／`zero_point` をどう定めるか、誰に敏感か、再訓練するかしないか、これがすべての学問だ。まず二大流派に分けよう：

- **PTQ（訓練後量子化、Post-Training Quantization）**：モデルを訓練し終えてから圧縮し、少量の**校正データ（calibration set）**だけで各層の数値範囲を推定し、勾配には触れない。安く、速く、元の訓練パイプラインを必要としない——GPTQ / AWQ / GGUFはすべてこの類で、オープンソース蒸留の主流だ。
- **QAT（量子化認識訓練、Quantization-Aware Training）**：訓練時に順伝播の中で**量子化の丸め誤差をシミュレートし**（逆伝播はStraight-Through Estimatorで勾配を直通させる）、重みが能動的に「圧縮に耐える」形に育つようにする。品質の上限は最も高いが、完全な訓練パイプラインが必要で、最も高価だ。**QLoRA（bitsandbytes NF4、Dettmers et al. 2023）** は賢い折衷案だ：4-bit NormalFloatの土台を凍結し、LoRAだけを訓練することで、QATに近い効果をファインチューニングの価格だけで得る。

認識しておくべき四つのPTQ手法は、それぞれ異なる痛点を解いている：

| 手法 | 一言の原理 | 解く痛点 | 典型ビット |
|---|---|---|---|
| **GPTQ**（Frantar 2022）| 層ごとに、二階（Hessian/OBQ）情報を用い、一列一列量子化して即時に誤差を補償する | 純重み低ビットの精度崩壊 | W4/W3 |
| **AWQ**（Lin 2023）| **活性値**から約1%の「顕著な重みチャネル」を見つけ出し保護スケーリングを掛け、残りはそのまま圧縮 | 重要な重みが一律に圧縮されて壊れる | W4 |
| **SmoothQuant**（Xiao 2022）| 活性の外れ値の「難しさ」をチャネルごとに一部重みへ移し、両側を均す | LLM活性の外れ値がW8A8を崩壊させる | W8A8 |
| **GGUF k-quants**（llama.cpp）| ブロック（super-block）単位で量子化、各ブロックが独立のscale、`_K_M/_S/_L` で異なるビットを混合 | エッジ・単機デプロイ、体積／品質のバランス | Q2_K–Q6_K |

混同されやすいいくつかの肝心な点：**①「重みだけ圧縮」vs「重み＋活性も圧縮」は二つの問題だ**——GPTQ/AWQ/GGUFは多くの場合重みだけを圧縮し（W4A16、活性は依然FP16）、SmoothQuant / FP8でようやく活性に触れる（W8A8）。なぜならLLMの活性には**外れ値（outlier）**があり、重みよりずっと圧縮しにくいからだ。**②校正データの代表性が成否を決める**：PTQの範囲推定はあの数百本の校正コーパスにすべてを頼る。校正セットと実際のトラフィックの分布がひとたびずれれば、低ビットは横転する——`imatrix` の本質はより念入りな校正だ（重要度で加重して守るべき重みを選ぶ）。**③FP8（E4M3/E5M2）は「より小さなINT8」ではない**：それは浮動小数点のダイナミックレンジを保持し、Hopper/Blackwellでネイティブのハードウェアサポートがあり、活性の外れ値に対してINT8よりはるかに寛容で、新世代の訓練／推論量子化の方向だ。**④INT4重み + FP8 KV Cacheが現在のエッジ長文の黄金の組み合わせだ**——重みを圧縮し切って省くのはモデル本体のVRAMで、KVをFP8でダイナミックレンジを保って省くのは「長文脈のあの溢れるメモリ」だ。

> 💡 ひとこと助言
> 君と語り合って一席。**量子化は「不可逆圧縮」の芸術だ——凡手は最小に圧縮することを追い求め、達人は同じ大きさの中に最も多くの魂を保つことを追い求める。** 「ワンクリックで全部4-bitに圧縮」と「アテンション層はQ8で守り、MLPはQ4に圧縮し、さらにimatrixで校正する」、最終的なファイルサイズはたかだか数百MBしか違わないかもしれないが、動かしたときの「知能指数」はまるで別人だ。違いはツールにあるのではない——ツールはみな同じ一式のオープンソース部品だ——違いはお前が**モデル内部のどこが関節で、どこが倉庫かを分かっているかどうか**だ。これは蒸留チェーン全体の縮図でもある：採集、KL方向、CoT重み付けから量子化まで、各ステップのツールはみな公開され無料だが、本当に希少なのは**「各ステップで何を守るべきか、何を犠牲にできるかを知る」判断力**だ。技術が民主化された時代、敷居はツールの入手にあるのではなく、センスの高低にある。

**蒸留方法論のスペクトル（コスト vs 神似度、一表で全章を束ねる）**：

| 段階 | 手法 | 教師から何を得るか | 神似度 | コスト／敷居 | クローズドソース教師に対して |
|---|---|---|---|---|---|
| ① | 純 SeqKD / SFT | 教師の出力テキスト（ハードターゲット）| 形は似ても神は似ず | 最低 | 可能だがToSを踏む |
| ② | SFT + CoT 重み付け + 軌跡反転/言い換え | テキスト + 推論連鎖 | 過学習を緩和、第一線の主力 | 中 | 可能だがToSを踏む |
| ③ | 選好蒸留 / DPO（場面5）| 教師の「AはBより優れる」順位 | 文体と判断尺度を盗む | 中高 | 複数回サンプリングが必要 |
| ④ | on-policy / GKD（場面3、reverse-KL）| 教師の生徒リアルタイム出力への採点 | 神似に最も近い | 高（教師への高頻度アクセス）| レート制限/検知に衝突 |
| ⑤ | 完全 logit 蒸留（場面1）| 教師の完全な確率分布 | 上限 | logitsが必要 | クローズドソースはそもそも出さない |

下に行くほど神似に近づき、また高価になり、防御の盾に衝突する——この表が下の解説の骨格である。

> 🔍 掘り下げ解説──蒸留の完全な方法論スペクトルと、「なぜ安い道が最も多く選ばれるか」
> 全章を繋ぎ合わせると、クローズドソースの最前線モデルに対する蒸留の方法論は、実は一道の**「コスト vs 神似度」のスペクトル**であり、安いものから高いものへとはっきり並んでいる：**① 純 SeqKD/SFT**（APIテキストでハードラベルのファインチューニング、最も安い、形は似ても神は似ず）→ **② SFT + CoT 重み付け + 軌跡反転/意味的言い換え**（中程度のコスト、過学習を緩和、第一線の主力）→ **③ DPO/棄却サンプリング**（選好で文体と判断をアライメント蒸留、やや高価）→ **④ on-policy 蒸留（reverse KL / GKD）**（神似に最も近いが、教師への高頻度アクセスが必要）→ **⑤ 完全 logit 蒸留**（最強だが、logitsが必要で、クローズドソースAPIはそもそも出さない）。現実はこうだ：**このスペクトルは右に行くほど強く、また高価になり、防御の盾に衝突する**。そして第1章のあの「リアルタイム検知 + 格下げ」が、まさに右側の数本の道のコストを天井まで押し上げる——教師への高頻度アクセスは検知を引き起こし、格下げによって汚染される。だからお前がHuggingFaceで見る「Claude distilled」の圧倒的多数がスペクトル最左の①②に留まっているのは、作者が③④⑤を知らないからではなく、**経済勘定と防御の盾が力を合わせて、彼らを最も安い道へと押し戻したからだ**。この一点を理解すれば、お前は分かる：**これらのオープンソースの影武者の「形は似ても神は似ず」は、技術の失敗ではなく、制約条件下での理性的な選択なのだ——そして蒸留の上限を本当に決めるのは、そもそもアルゴリズムではなく、お前が（合法的に、低コストで）教師のあの「躊躇の分布」を持続的に手に入れられるかどうかなのだ。**

## 本章のまとめ

- **ソフトラベル > ハードラベル**：盗むのは「センス（確率分布／暗黙知）」である；Hinton損失 `α·CE + (1-α)·T²·KL` が骨幹（`T²` は必ず補償、暗黙知 = one-hotが持たないあの一段のエントロピー）、クローズドソースAPIがlogitsを出さないことがSeqKDを強いる。
- **KL方向が人格を決める**：forward KL（`P` で加重、mode-covering、お人好し）vs reverse KL（`Q` で加重、mode-seeking、専門家）；小モデルが大モデルを学ぶ場合はreverse KLを優先する。
- **系列レベル vs on-policy**：トークンレベルのteacher-forcingには露出バイアスがある；Kim & Rush SeqKDは教師の文全体をハードターゲットにし、GKD/MiniLLMは生徒にまず生成させて教師が採点する（on/off-policy × forward/reverse の二つの直交したつまみ）——「誰の軌跡の上で学ぶか」は「どれだけ似ているか」より上限を決める。
- **CoT重み付け + 軌跡反転 + 理由蒸留（Distilling step-by-step）+ 意味的言い換え**：丸暗記の近道を塞ぎ、思考を重みの中に内面化する；だが摂動は意味的真理値を歪めてはならず、かつ「output→prompt 反転」式の軌跡反転は抽出の一線を踏む。
- **選好蒸留 / DPO**：logitsが手に入らなくても「AはBより優れる」の順位からセンスを盗める；`β`-KLはreverse方向を暗に含む——だが同じ経路は逆に「無センス」も伝えられる（脱検閲）。
- **語彙横断蒸留（ULD）** と **量子化の科学**：PTQ（GPTQ/AWQ/SmoothQuant/GGUF k-quants）vs QAT/QLoRA；重みだけ圧縮 vs 活性も含めて、校正データの代表性、FP8≠小さなINT8、INT4重み + FP8 KVのエッジ黄金の組み合わせ。
- **方法論スペクトル（①→⑤）**：コストが高いほど神似だが、防御の盾が右側を天井まで押し上げる——これこそオープンソースの影武者が「形似」に留まる真因だ。

原理は語り尽くした。だが原理の外には、まだ「企業級」を謳う**自動化蒸留パイプライン**を持ち出す者がいる、五大モジュール、四十ステップ。次章ではそれを丸ごと一本テーブルに載せる：学ぶべきは学び、警戒すべきは警戒する。
