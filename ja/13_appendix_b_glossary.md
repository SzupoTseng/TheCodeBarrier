# 付録B　用語とツール早見表

> 本書に登場する技術用語を、テーマごとにまとめた一枚の「デコーダー」。これらは**すべて実在し、公開されている技術**であり、付録 A の「未検証のモデル名」とは異なる——これらを押さえれば、あらゆる蒸留／ローカル運用の宣伝文句が、結局のところ何を言っているのかを聞き分けられるようになる。

## B.1　蒸留の中核概念

| 用語 | 一言での説明 | 登場章 |
|---|---|---|
| **知識蒸留（Knowledge Distillation, KD）** | 大きな教師モデルが能力を小さな生徒モデルに引き継がせる | 全書 |
| **教師 / 生徒（Teacher / Student）** | 学ばれる大モデル / 学びに来る小モデル | 第 4、5 章 |
| **ハードラベル（Hard Target）** | 最終的な答えだけを学ぶ | 第 4 章 |
| **ソフトラベル / Logits（Soft Target）** | 教師の完全な確率分布（「暗黙知」）を学ぶ | 第 4 章 |
| **KL 散度（KL-Divergence）** | 二つの確率分布がどれだけ離れているかを測る数学ツール | 第 4、5 章 |
| **思考連鎖（Chain-of-Thought, CoT）** | モデルが一歩ずつ推論していく過程 | 第 2、4、5 章 |
| **CoT アライメント損失** | 思考過程に重みを与え、生徒に「考えてから答える」を強いる | 第 4、5 章 |
| **軌跡反転（Trace Inversion）** | 推論ステップを撹乱／逆順にし、生徒の丸暗記を防ぐ | 第 2、4 章 |
| **意味的言い換え（Synthetic Paraphrasing）** | 高温サンプリングでデータを書き換え、汎化を広げ過学習を防ぐ | 第 4、5 章 |
| **過学習（Overfitting）** | 訓練データを丸暗記し、場面が変わると崩れる | 第 4 章 |
| **abliteration** | 重みの中から「拒否」方向を消し去り、脱検閲を行う | 第 2 章 |

## B.2　訓練とファインチューニング

| 用語 | 説明 |
|---|---|
| **SFT（指示ファインチューニング）** | 「指示→応答」のペアで教師ありファインチューニングする |
| **QLoRA** | 量子化 + 低ランク適応の効率的なファインチューニング手法（VRAM 節約）|
| **全パラメータファインチューニング（Full FT）** | モデルの全重みを更新する（最も徹底的で、最もリソースを食う）|
| **DeepSpeed ZeRO-3** | 重み／勾配／オプティマイザ状態を分割し、メモリを節約して大モデルを動かす |
| **混合精度（BF16/FP16）** | より低い精度で訓練を加速しつつ、安定性も両立する |
| **勾配クリッピング / コサインアニーリング / Early Stopping** | 訓練の崩壊を防ぎ、収束を安定させる三つのガードレール |
| **YaRN + RoPE Base 拡張** | モデルのコンテキストウィンドウを 8K から 1M まで段階的に引き伸ばす技術 |
| **FlashAttention-3** | 効率的なアテンション計算で、メモリを節約し長系列を加速する |

## B.3　量子化とデプロイ

| 用語 | 説明 |
|---|---|
| **量子化（Quantization）** | 重みを高精度から低精度へ圧縮し、サイズ／速度と引き換えにする |
| **GGUF** | ローカル運用の主流モデル形式（llama.cpp/Ollama/LM Studio）|
| **Q4_K_M / Q5_K_M / Q8_0** | よく使う量子化レベル。数字が大きい=精確/大、K_M=混合精度 |
| **混合量子化** | アテンション層は高精度を保ち、MLP 層を低精度に圧縮する |
| **KV Cache** | コンテキスト用メモリ。長文における真のボトルネック |
| **KV Cache 量子化（FP8/INT8）** | コンテキスト用メモリを圧縮し、長文のポテンシャルを解き放つ |
| **MoE（混合エキスパート）** | 多数のエキスパートのうち毎回少数のみを起動する。帯域幅律速マシンの救世主 |
| **起動パラメータ vs 総パラメータ** | 前者が速度を、後者が知能の上限を決める |

## B.4　ハードウェア（NVIDIA DGX Spark）

| 用語 | 説明 |
|---|---|
| **DGX Spark** | NVIDIA のデスクトップ型「個人 AI スーパーコンピュータ」|
| **GB10（Grace Blackwell）** | DGX Spark に搭載されたスーパーチップ |
| **UMA（統一メモリアーキテクチャ）** | CPU と iGPU が同一プールのメモリを共有する |
| **128GB LPDDR5X（~600 GB/s）** | 容量は巨大だが、帯域幅は単体 GPU の HBM/GDDR に及ばない |
| **容量 vs 帯域幅の鉄則** | 容量は「動かせるか」を、帯域幅は「どれだけ速いか」を決める |
| **numactl --interleave=all / mlock** | メモリをロックし、チャネルをインターリーブして UMA の帯域幅を絞り尽くす |
